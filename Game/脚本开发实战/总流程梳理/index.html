    二、核心模块实现逻辑#

    2.1 多线程架构设计#

    易语言模式迁移:

    全局线程信息数组 → Python类实例列表
- 每个线程对应一个游戏窗口
- 独立的大漠对象
- 独立的配置节点
- 独立的鼠标漂移修正参数

    线程生命周期:

    1. 初始化: 创建大漠对象 → 加载字库 → 绑定窗口
    2. 运行: 状态机循环(读取任务→寻路→战斗→完成)
    3. 暂停/恢复: 通过标志位控制
    4. 停止: 释放资源

    2.2 鼠标漂移解决方案(核心技术)#

    问题分析: 后台绑定VNC窗口时,绝对坐标移动存在系统级偏差

    解决思路:

    原始移动: MoveTo(x, y) → 实际落点存在随机漂移

修正方案:
1. 初次移动: 绝对移动到目标点附近
2. 位置校准: 
   - 字库识别法: 识别鼠标光标特征字
   - 图像匹配法: 模板匹配鼠标指针图片
3. 误差计算: Δx = 目标x - 实际x, Δy = 目标y - 实际y
4. 循环修正: 相对移动(Δx, Δy) 直到误差<阈值
5. 自学习: 记录不同区域的修正值,下次直接预补偿

    关键点:

    • 鼠标在不同屏幕区域(左/中/右,上/中/下)漂移规律不同
    • 需要建立分区校正模型
    • 新区卡顿和老区流畅需要自适应调整修正策略

    2.3 图像识别系统#

    三层识别机制:

    第1层: 快速找图
- 大漠找图
- 用于UI元素检测(按钮/对话框)
- 特点:速度快,精度一般

第2层: 精确找图  
- OpenCV模板匹配
- 用于NPC/物品识别
- 特点:可调阈值,支持多图匹配

第3层: OCR文字识别
- 字库查找(快但需要预建字库)
- PaddleOCR/Tesseract(慢但通用)
- 用于任务内容/坐标/地图名解析

    识别容错策略:

    • 多图备选: “图片A|图片B|图片C” 匹配任一即成功
    • 区域限定: 动态计算识别区域,减少误判
    • 超时重试: 识别失败→延时→重试→超时降级处理

    2.4 地图导航系统#

    核心逻辑:

    code

    复制

    插入

    新建文件

    保存

    应用

    1. 获取当前位置
   - OCR识别左上角"地图名[x,y]"
   - 或内存读取坐标数据

2. 路径规划
   - 判断是否跨地图 → 调用传送(驿站/飞行旗)
   - 同地图 → 计算小地图点击点

3. 坐标换算
   - 游戏坐标 → 屏幕像素坐标
   - 公式: screen_x = 小图_x + (游戏_x / 地图最大_x * 小图宽度)
          screen_y = 小图_y - (游戏_y / 地图最大_y * 小图高度)

4. 寻路等待
   - 持续检测位置是否变化
   - 连续N次坐标不变 → 判定到达

    跨地图处理:

    • 预定义地图节点配置(传送点坐标、地图连接关系)
    • 飞行旗/驿站优先级策略
    • 未知地图的兜底方案(步行/问路)

    三、关键难点与解决方案#

    3.1 配置隔离与多开#

    问题: 多个游戏窗口需要独立配置,避免参数冲突

    方案:

    • 配置文件结构: config.ini 按"窗口_1/窗口_2"分节点
    • 运行时通过窗口索引动态读取对应节点
    • 全局配置(如分辨率) + 窗口配置(如VNC名称、鼠标修正值)

    3.2 对话框自动化#

    难点: 游戏对话框内容动态,按钮位置不固定

    方案:

    1. 界面判定: 检测对话框标识图片(如"关闭按钮")
    2. 按钮识别: OCR识别按钮文字(“接受/拒绝/取消”)
    3. 相对点击: 基于对话框位置计算按钮相对坐标
    4. 超时处理: 对话框未弹出→点击NPC→重试→超时放弃

    3.3 NPC遮挡处理#

    场景: NPC被其他玩家或物体遮挡,无法正常点击

    处理策略:

    1. 屏蔽功能: Alt+H 屏蔽其他玩家
    2. 视角拉近: 点击NPC附近位置,拉近摄像机
    3. Ctrl机制: Ctrl+点击NPC,显示所有重叠NPC列表,OCR识别目标NPC名并点击
    4. 多角度尝试: 从不同方位接近并点击

    3.4 新老区性能适配#

    问题: 新区画面卡顿,老区流畅运行

    自适应策略:

    • 检测机制: 统计位置变化频率,卡顿检测(连续N次位置不变)

    • 新区优化

      :

      • 增大延时参数
      • 增大识别容错阈值
      • 减少高频循环
      • 跳过部分校准步骤

    • 老区保持: 使用默认的高性能参数

    六、总结与建议#

    核心设计理念:

    1. 模块化: 将大漠插件的各个功能封装为独立Python模块
    2. 状态机: 复杂流程使用状态机管理,避免嵌套if-else
    3. 容错性: 每个关键步骤都应有重试机制和超时处理
    4. 可配置: 参数外置,支持不同游戏/区域/性能需求

    易语言→Python迁移要点:

    • 易语言"子程序" → Python函数/方法
    • 易语言"自定义数据类型" → Python类
    • 易语言"全局数组" → Python列表/字典
    • 易语言"判断循环" → Python while循环

    最大挑战: 鼠标漂移问题需要重点攻克,建议先做原型验证

    1.1 打图任务整体流程 还是不懂这个流程，是都放在一个while里面吗 请给我一段模拟该任务的python代码,让我理解一下

    
"""
打图任务执行流程示例
展示状态机在一个while循环中的工作方式
"""

class DaotuTask:
    """打图任务类"""
    
    def __init__(self, window_index):
        self.window_index = window_index  # 窗口索引
        self.current_state = "读取当前任务"  # 当前状态
        self.task_info = None  # 任务信息(字典)
        
        # 状态列表(定义所有可能的状态)
        self.states = [
            "读取当前任务",      # 状态1
            "接取任务",          # 状态2
            "选择传送方式",      # 状态3
            "跨地图导航",        # 状态4
            "同地图寻路",        # 状态5
            "点击强盗NPC",       # 状态6
            "战斗循环",          # 状态7
            "任务完成",          # 状态8
        ]
    
    def run(self):
        """任务主循环 - 只有一个while!"""
        print(f"🚀 窗口{self.window_index} 打图任务开始...")
        
        # ============================================================
        # 核心: 只有一个while循环
        # ============================================================
        while True:  # 👈 只有一个while!
            
            try:
                # ====================================================
                # 状态机: 根据current_state执行对应逻辑
                # ====================================================
                
                # ----------------------------------------
                # 状态1: 读取当前任务
                # ----------------------------------------
                if self.current_state == "读取当前任务":
                    print(f"📍 当前状态: {self.current_state}")
                    
                    # 尝试读取任务(从任务追踪栏或任务面板)
                    task = self._read_task_from_ui()
                    
                    if task is not None:
                        # 👉 任务读取成功,保存任务信息
                        self.task_info = task
                        print(f"✅ 任务读取成功:")
                        print(f"   怪物名: {task['monster_name']}")
                        print(f"   地图名: {task['map_name']}")
                        print(f"   坐标: ({task['x']}, {task['y']})")
                        
                        # 👉 切换到下一个状态: 选择传送方式
                        self.current_state = "选择传送方式"
                    else:
                        # 👉 没有任务,需要接取任务
                        print("❌ 当前没有任务,需要接取任务")
                        self.current_state = "接取任务"
                
                # ----------------------------------------
                # 状态2: 接取任务
                # ----------------------------------------
                elif self.current_state == "接取任务":
                    print(f"📍 当前状态: {self.current_state}")
                    
                    # 1. 屏蔽其他玩家
                    self._shield_other_players()
                    
                    # 2. 寻路到店小二
                    self._move_to_npc("店小二")
                    
                    # 3. 点击NPC对话
                    if self._click_npc_and_talk("店小二"):
                        # 4. 接取任务
                        print("📋 正在接取打图任务...")
                        time.sleep(2)
                        
                        # 5. 返回状态1重新读取任务
                        self.current_state = "读取当前任务"
                    else:
                        # 点击失败,继续尝试
                        print("⚠️  点击NPC失败,重试...")
                
                # ----------------------------------------
                # 状态3: 选择传送方式
                # ----------------------------------------
                elif self.current_state == "选择传送方式":
                    print(f"📍 当前状态: {self.current_state}")
                    
                    map_name = self.task_info['map_name']
                    
                    # 根据目标地图选择传送方式
                    if map_name == "建邺城":
                        print(f"🎯 目标地图: {map_name} - 使用长安酒店飞行旗")
                        if self._use_flying_flag("长安酒店"):
                            # 飞行成功,跳过跨地图导航,直接同地图寻路
                            self.current_state = "同地图寻路"
                    
                    elif map_name == "大唐国境":
                        print(f"🎯 目标地图: {map_name} - 使用驿站传送")
                        self.current_state = "跨地图导航"
                    
                    else:
                        print(f"🎯 目标地图: {map_name} - 使用飞行符")
                        if self._use_flying_charm(map_name):
                            self.current_state = "同地图寻路"
                
                # ----------------------------------------
                # 状态4: 跨地图导航
                # ----------------------------------------
                elif self.current_state == "跨地图导航":
                    print(f"📍 当前状态: {self.current_state}")
                    
                    # 使用驿站传送到目标地图
                    if self._teleport_by_yizhan(self.task_info['map_name']):
                        print(f"✅ 已传送到 {self.task_info['map_name']}")
                        # 👉 传送成功,进入同地图寻路
                        self.current_state = "同地图寻路"
                    else:
                        print("❌ 传送失败,重试...")
                
                # ----------------------------------------
                # 状态5: 同地图寻路
                # ----------------------------------------
                elif self.current_state == "同地图寻路":
                    print(f"📍 当前状态: {self.current_state}")
                    
                    # 移动到任务坐标附近
                    target_x = self.task_info['x']
                    target_y = self.task_info['y']
                    
                    if self._move_to_position(target_x, target_y):
                        print(f"✅ 已到达目标坐标: ({target_x}, {target_y})")
                        # 👉 到达目标,点击强盗
                        self.current_state = "点击强盗NPC"
                    else:
                        print("❌ 寻路失败,重试...")
                
                # ----------------------------------------
                # 状态6: 点击强盗NPC
                # ----------------------------------------
                elif self.current_state == "点击强盗NPC":
                    print(f"📍 当前状态: {self.current_state}")
                    
                    monster_name = self.task_info['monster_name']
                    
                    # 尝试点击强盗
                    if self._click_monster(monster_name):
                        # 👉 成功触发对话,进入战斗
                        if self._check_dialog_appear():
                            print("🗣️  对话框已弹出,即将进入战斗...")
                            time.sleep(1)
                            self.current_state = "战斗循环"
                        else:
                            # 对话框未弹出,可能被遮挡
                            print("⚠️  对话框未弹出,尝试Ctrl+点击")
                            self._ctrl_click_monster(monster_name)
                    else:
                        print("⚠️  点击强盗失败,重试...")
                
                # ----------------------------------------
                # 状态7: 战斗循环
                # ----------------------------------------
                elif self.current_state == "战斗循环":
                    print(f"📍 当前状态: {self.current_state}")
                    
                    # 持续检测战斗状态
                    while self._check_in_battle():
                        print("⚔️  战斗中...")
                        
                        # 血量检测
                        current_hp = self._get_current_hp()
                        if current_hp < 30:
                            print(f"❤️  血量低({current_hp}%),使用金创药")
                            self._use_item("金创药")
                        else:
                            # 执行攻击
                            self._select_skill("横扫千军")
                            self._attack_target()
                        
                        time.sleep(1)  # 等待回合
                    
                    # 战斗结束
                    print("🎉 战斗结束!")
                    
                    # 检查战斗结果
                    battle_result = self._check_battle_result()
                    if battle_result == "胜利":
                        print("✅ 战斗胜利!")
                        self.current_state = "任务完成"
                    elif battle_result == "失败":
                        print("💀 战斗失败,需要复活...")
                        self._revive()
                        self.current_state = "点击强盗NPC"
                    else:
                        print("⚠️  战斗异常,重试...")
                        self.current_state = "点击强盗NPC"
                
                # ----------------------------------------
                # 状态8: 任务完成
                # ----------------------------------------
                elif self.current_state == "任务完成":
                    print(f"📍 当前状态: {self.current_state}")
                    
                    # 关闭对话框
                    self._close_dialog()
                    
                    # 检查是否还有后续任务
                    if self._has_next_task():
                        print("📋 还有后续任务,继续...")
                        # 👉 返回状态1读取新任务
                        self.current_state = "读取当前任务"
                    else:
                        print("🎊 当前任务链完成!")
                        # 可以在这里选择:
                        # 1. 继续循环,等待新任务
                        # 2. 返回主菜单选择其他任务
                        # 这里我们选择循环等待
                        self.current_state = "读取当前任务"
                
                # ----------------------------------------
                # 状态处理完成,延时后进入下一轮循环
                # ----------------------------------------
                print("=" * 50)
                time.sleep(0.5)  # 避免CPU占用过高
                
            except Exception as e:
                print(f"❌ 状态 '{self.current_state}' 执行出错: {e}")
                # 可以选择:
                # 1. 重试当前状态
                # 2. 返回初始状态
                # 3. 记录日志并退出
                time.sleep(2)
                # 返回初始状态继续
                self.current_state = "读取当前任务"
