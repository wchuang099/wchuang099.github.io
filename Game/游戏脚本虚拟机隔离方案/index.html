    脚本隔离方案

    • 方案 1：VNC 协议
    • 方案 2：虚拟机 + 双头盒子（硬件）
    • 方案 3：大漠识别 + 自研通信协议（软件实现）

    虚拟机游戏脚本隔离方案#

    背景：大漠能抓虚拟机画面，但无法跨系统直接发送键鼠，需中间层解决输入问题。

    方案 1：VNC 协议#

    1-1 VMware 自带 VNC（简单零成本） VMware 开启 VNC 服务端 宿主机用 VNC 客户端连接 大漠绑定 VNC 客户端窗口 实现：取图 + 后台键鼠输入 成熟稳定，最易落地。

    1-2 自研 VNC 插件（高自由度，类似unify）

    VNC 层：直接连接 VM RFB 流 → 收画面 + 发键鼠 脚本层：封装图色/OCR/点击/按键接口，像大漠一样调用

    方案 2：虚拟机 + 双头盒子（硬件）#

    大漠绑定 VM 窗口取图 双头盒子以物理 HID 模拟键鼠 零封号风险但成本高；适合大量机器。

    方案 3：大漠识别 + 自研通信协议（软件实现）#

    宿主机：大漠负责识别画面 通过自定义协议发送键鼠指令到虚拟机 虚拟机需安装一个 Agent 用来接收指令并调用 SendInput 软件成本最低、扩展性强，但需在 VM 内运行 Agent。
