    1. 介绍#

    1.1 什么是 OpenClaw#

    OpenClaw 是一个开源的自托管 AI 网关，可以将 Discord、Telegram、WhatsApp、微信、飞书等聊天平台连接到 AI 模型。你在自己的机器上运行一个 Gateway 进程，就可以从任何地方通过消息与 AI 助手对话。

    1.2 系统要求#

    • Windows 10 / 11
    • Node.js 24（推荐）或 Node 22.16+（安装脚本会自动处理）
    • AWS 账号，已开通 Bedrock 模型访问权限
    • 稳定的网络连接

    1.3 架构与运行原理#

    两种运行模式#

    模式 1：Gateway 模式（服务端 + 客户端，推荐）

    ┌─────────────────────────────────────────────────┐
│  Gateway（长期运行的服务进程）                      │
│  - 监听 127.0.0.1:18789                         │
│  - 管理会话、记忆、频道路由                        │
│  - 持有 AWS 凭证，负责调用模型                     │
│  - 提供 Web 控制台、WebSocket 接口                │
└─────────────────────────────────────────────────┘
        ↑ 连接
┌───────────────────┐  ┌───────────────────┐
│ openclaw agent    │  │ Web 控制台         │
│ (一次性客户端)     │  │ Telegram/微信等    │
└───────────────────┘  └───────────────────┘

    模式 2：嵌入式模式（–local / EMBEDDED FALLBACK）

    ┌─────────────────────────────────────────────────┐
│  openclaw agent --local                          │
│  - 临时启动一个完整的 agent 运行时                 │
│  - 自己加载插件、自己调用模型                      │
│  - 用完就退出，不保留后台进程                      │
│  - 需要当前 shell 有 AWS 凭证                     │
└─────────────────────────────────────────────────┘

    关键概念#

    • Gateway 不是守护进程：不会自动运行，需要手动启动或通过计划任务自动启动
    • EMBEDDED FALLBACK：当 Gateway 没在运行时，openclaw agent 会自动回退到嵌入式模式完成调用
    • 计划任务：通过 openclaw gateway install（需管理员权限）注册 Windows 计划任务，实现登录时自动启动 Gateway

    如何选择#

    场景 推荐方式
    偶尔用一下，问个问题 不开 Gateway，直接 openclaw agent --local -m "问题"
    频繁使用 / 用 Web 界面 / 接 Telegram 等频道 开 Gateway 后台运行
    只在终端偶尔问问 不开 Gateway 也行，EMBEDDED FALLBACK 会自动兜底

    查看计划任务#

    # 命令行查看
schtasks /query /tn "OpenClaw Gateway" /v /fo LIST

# 用 openclaw 命令查看
openclaw gateway status

# 图形界面查看：Win+R → 输入 taskschd.msc → 找 "OpenClaw Gateway"

    2. 安装#

    2.1 运行安装脚本#

    打开 PowerShell（普通权限即可），执行：

    iwr -useb https://openclaw.ai/install.ps1 | iex

    脚本会自动完成以下操作：

    • 检测并安装 Node.js（通过 winget / Chocolatey / Scoop）
    • 全局安装 OpenClaw CLI
    • 启动交互式 onboarding 引导

    2.2 Onboarding 引导选择#

    步骤 推荐选择
    Personal-by-default 确认 Yes
    Setup mode QuickStart (recommended)
    Config handling（如有已有配置） Keep current values
    Model/auth provider Skip for now
    Default model Skip 或 Keep current
    Select channel Skip for now
    Search provider Skip for now
    Configure skills Yes（但依赖项选 Skip）
    各种 API Key 提示 全部 No
    Enable hooks Skip for now
    Hatch agent Hatch later

    2.3 安装 Bedrock 插件#

    OpenClaw 原生支持 Amazon Bedrock 作为模型 provider。在某些环境下（如 Windows 本地安装），可能需要手动安装 Bedrock 插件：

    openclaw plugins install @openclaw/amazon-bedrock-provider

    说明： 如果运行 openclaw config set 配置 Bedrock 时提示 “plugin not installed: amazon-bedrock”，则需要执行上述命令。如果没有提示，说明已内置支持，可跳过此步。

    2.4 配置 Bedrock#

    2.4.1 启用 Discovery 插件#

    openclaw config set plugins.entries.amazon-bedrock.config.discovery.enabled true
openclaw config set plugins.entries.amazon-bedrock.config.discovery.region us-east-1

    2.4.2 手动注册模型（关键步骤）#

    仅靠 discovery 插件不够，必须在配置文件 ~\.openclaw\openclaw.json 中手动添加 models.providers 部分：

    {
  "models": {
    "providers": {
      "amazon-bedrock": {
        "baseUrl": "https://bedrock-runtime.us-east-1.amazonaws.com",
        "api": "bedrock-converse-stream",
        "auth": "aws-sdk",
        "models": [
          {
            "id": "us.amazon.nova-pro-v1:0",
            "name": "Amazon Nova Pro",
            "reasoning": false,
            "input": ["text", "image"],
            "contextWindow": 300000,
            "maxTokens": 5120
          },
          {
            "id": "us.amazon.nova-lite-v1:0",
            "name": "Amazon Nova Lite",
            "reasoning": false,
            "input": ["text", "image"],
            "contextWindow": 300000,
            "maxTokens": 5120
          },
          {
            "id": "us.deepseek.r1-v1:0",
            "name": "DeepSeek R1",
            "reasoning": true,
            "input": ["text"],
            "contextWindow": 128000,
            "maxTokens": 8192
          }
        ]
      }
    }
  }
}

    2.4.3 设置默认模型#

    openclaw config set agents.defaults.model.primary "amazon-bedrock/us.amazon.nova-pro-v1:0"

    2.5 可用模型参考#

    模型 ID 说明 支持 Tool Use
    us.amazon.nova-pro-v1:0 Amazon Nova Pro，推荐日常使用 ?
    us.amazon.nova-lite-v1:0 Amazon Nova Lite（更便宜） ?
    us.amazon.nova-micro-v1:0 Amazon Nova Micro（最便宜） ?
    us.deepseek.r1-v1:0 DeepSeek-R1，推理增强 ?
    us.meta.llama4-maverick-17b-instruct-v1:0 Llama 4 Maverick ?

    注意事项：

    • Anthropic Claude 系列模型在中国大陆有地区限制，会报错 “Access to Anthropic models is not allowed from unsupported countries”
    • DeepSeek R1 不支持流式 Tool Use（会报 “This model doesn’t support tool use in streaming mode”），如需使用需关闭工具功能
    • 如果组织有 SCP（Service Control Policy）限制，可能无法直接调用 foundation model，需通过 inference profile 调用

    2.6 设置 AWS 凭证#

    临时设置（当前会话有效）#

    $env:AWS_ACCESS_KEY_ID = "你的AK"
$env:AWS_SECRET_ACCESS_KEY = "你的SK"
$env:AWS_REGION = "us-east-1"

    永久设置（重启后仍有效）#

    [Environment]::SetEnvironmentVariable("AWS_ACCESS_KEY_ID", "你的AK", "User")
[Environment]::SetEnvironmentVariable("AWS_SECRET_ACCESS_KEY", "你的SK", "User")
[Environment]::SetEnvironmentVariable("AWS_REGION", "us-east-1", "User")

    设置后需要重新打开 PowerShell 窗口才能生效。

    2.7 验证安装（可选）#

    安装配置完成后，可以用以下命令确认一切正常：

    # 确认 CLI 版本
openclaw --version

# 检查配置是否有问题（类似体检，会报告缺失配置、权限等）
openclaw doctor

# 列出当前可用模型，确认 Bedrock 模型已被识别
openclaw models list

    如果前面步骤都顺利完成且能正常对话，这些验证命令可以跳过。

    2.8 安装 Gateway 后台服务（推荐）#

    右键点击 PowerShell → “以管理员身份运行”，然后执行：

    openclaw gateway install

    这会创建 Windows 计划任务，登录时自动启动 Gateway 后台服务，无需手动保持窗口。

    前提： 必须先设置永久环境变量（见 2.6 节），否则后台服务读不到 AWS 凭证。

    3. 使用#

    3.1 启动 Gateway#

    在 PowerShell 中设置 AWS 凭证并启动 Gateway：

    $env:AWS_ACCESS_KEY_ID = "你的AK"
$env:AWS_SECRET_ACCESS_KEY = "你的SK"
$env:AWS_REGION = "us-east-1"
openclaw gateway run

    Gateway 是 OpenClaw 的核心服务，负责处理所有消息路由和模型调用。启动后保持窗口运行。

    3.2 发送消息#

    通过 Gateway（推荐）#

    另开一个 PowerShell 窗口，不需要设置 AWS 环境变量：

    openclaw agent --agent main -m "你好"

    本地直连模式（–local）#

    不经过 Gateway，直接调用模型。当前窗口必须有 AWS 环境变量：

    $env:AWS_ACCESS_KEY_ID = "你的AK"
$env:AWS_SECRET_ACCESS_KEY = "你的SK"
$env:AWS_REGION = "us-east-1"
openclaw agent --agent main -m "你好" --local

    两种方式对比#

    方式 凭证要求 说明
    不加 --local（走 Gateway） 只需 Gateway 窗口有环境变量 推荐，客户端窗口不需要凭证
    加 --local（本地直连） 当前窗口必须有 AWS 环境变量 每个窗口都要单独设置凭证

    永久设置环境变量（免去每次设置）#

    [Environment]::SetEnvironmentVariable("AWS_ACCESS_KEY_ID", "你的AK", "User")
[Environment]::SetEnvironmentVariable("AWS_SECRET_ACCESS_KEY", "你的SK", "User")
[Environment]::SetEnvironmentVariable("AWS_REGION", "us-east-1", "User")

    设置后重新打开所有 PowerShell 窗口即可生效，以后不用再手动设置。

    3.3 使用 TUI 交互界面#

    openclaw tui

    TUI 提供终端内的交互式聊天体验，支持多轮对话。

    3.4 使用 Web 控制台#

    Gateway 启动后，运行以下命令打开 Web 控制台（会自动带上认证 Token）：

    openclaw dashboard

    注意： 直接在浏览器访问 http://127.0.0.1:18789 会报 Unauthorized，必须通过 openclaw dashboard 命令打开才能自动完成认证。

    3.5 聊天斜杠命令#

    在对话中可以使用以下命令：

    命令 作用
    /new 开始新会话（清空上下文）
    /reset 重置当前会话
    /compact 手动压缩对话历史（节省 token）
    /compact 重点保留API设计决策 带指引的压缩
    /status 查看当前状态、token 用量
    /usage tokens 每次回复后显示 token 消耗
    /usage off 关闭 token 显示
    /context list 查看系统提示词中包含了什么

    3.6 切换模型#

    openclaw config set agents.defaults.model.primary "amazon-bedrock/deepseek.r1-v1:0"

    切换后需要重启 Gateway 才能生效。

    3.7 查看可用模型#

    openclaw models list

    4. 节省 Token 策略#

    4.1 按场景选择模型#

    场景 推荐模型 原因
    日常闲聊、简单问答 Nova Micro / Nova Lite 便宜，响应快
    复杂任务、代码生成 DeepSeek V3.2 性价比高
    深度推理、数学逻辑 DeepSeek R1 推理能力强

    4.2 会话管理#

    及时开新会话#

    完成一个话题后用 /new 开始新会话，避免无关上下文累积消耗 token。

    手动压缩#

    对话过长时用 /compact 压缩历史，只保留摘要：

    /compact

    或带指引压缩：

    /compact 只保留关于数据库设计的讨论

    设置空闲自动重置#

    编辑配置文件 ~\.openclaw\openclaw.json，添加：

    {
  "session": {
    "reset": {
      "idleMinutes": 60
    }
  }
}

    空闲 60 分钟后自动开始新会话，避免长时间不用的会话继续累积上下文。

    4.3 减少系统提示词开销#

    精简 Skills#

    只安装需要的 skills，每个 skill 的描述都会注入到系统提示词中，占用 token。

    精简 Bootstrap 文件#

    工作区中的 AGENTS.md、SOUL.md、IDENTITY.md、USER.md 等文件内容会注入提示词，保持简洁。

    限制 Bootstrap 大小#

    {
  "agents": {
    "defaults": {
      "bootstrapMaxChars": 8000,
      "bootstrapTotalMaxChars": 30000
    }
  }
}

    4.4 控制工具输出大小#

    大型工具输出会消耗大量 token，可以限制最大字符数：

    {
  "agents": {
    "defaults": {
      "contextLimits": {
        "toolResultMaxChars": 5000
      }
    }
  }
}

    4.5 降低图片 Token 消耗#

    如果经常发送截图，降低图片分辨率可以减少 vision token：

    {
  "agents": {
    "defaults": {
      "imageMaxDimensionPx": 800
    }
  }
}

    4.6 用便宜模型做 Compaction#

    对话压缩（compaction）本身也消耗 token，可以用便宜的模型来做：

    {
  "agents": {
    "defaults": {
      "compaction": {
        "model": "amazon-bedrock/us.amazon.nova-lite-v1:0"
      }
    }
  }
}

    4.7 监控用量#

    • 在对话中输入 /status 查看当前会话 token 消耗
    • 输入 /usage tokens 开启每次回复的 token 统计
    • 在 AWS Console → Bedrock → Model invocation logging 查看整体用量和费用

    5. 维护#

    5.1 更新 OpenClaw#

    openclaw update

    或重新运行安装脚本：

    iwr -useb https://openclaw.ai/install.ps1 | iex

    5.2 健康检查#

    openclaw doctor

    5.3 查看日志#

    openclaw logs

    日志文件位置：%TEMP%\openclaw\openclaw-<日期>.log

    5.4 会话清理#

    预览可清理的会话：

    openclaw sessions cleanup --dry-run

    执行清理：

    openclaw sessions cleanup --enforce

    配置自动清理（编辑 ~\.openclaw\openclaw.json）：

    {
  "session": {
    "maintenance": {
      "mode": "enforce",
      "pruneAfter": "30d",
      "maxEntries": 500
    }
  }
}

    5.5 Gateway 服务管理#

    # 前台启动（需保持窗口）
openclaw gateway run

# 查看状态
openclaw gateway status

# 安装为后台服务（需管理员权限 PowerShell，开机自动启动）
openclaw gateway install

# 重启
openclaw gateway restart

    后台运行说明： openclaw gateway install 会创建 Windows 计划任务（Scheduled Task），Gateway 在后台运行，开机自动启动，无需保持窗口打开。必须用管理员权限的 PowerShell 执行（右键 → 以管理员身份运行）。

    重要： 后台服务需要永久环境变量才能读到 AWS 凭证。临时的 $env: 变量对后台服务无效。

    5.6 备份配置#

    配置文件位置：~\.openclaw\openclaw.json

    Copy-Item "$env:USERPROFILE\.openclaw\openclaw.json" "$env:USERPROFILE\.openclaw\openclaw.json.manual-backup"

    5.7 卸载#

    卸载 CLI：

    npm uninstall -g openclaw

    完全清除所有数据：

    Remove-Item -Recurse -Force "$env:USERPROFILE\.openclaw"

    6. 配置文件详解#

    配置文件路径：~\.openclaw\openclaw.json

    以下是当前实际配置的逐项解读：

    6.1 models（模型注册）#

    {
  "models": {
    "providers": {
      "amazon-bedrock": {
        "baseUrl": "https://bedrock-runtime.us-east-1.amazonaws.com",
        "api": "bedrock-converse-stream",
        "auth": "aws-sdk",
        "models": [...]
      }
    }
  }
}
    字段 说明
    baseUrl Bedrock 运行时端点 URL，格式为 https://bedrock-runtime.{region}.amazonaws.com
    api API 协议，固定为 bedrock-converse-stream（流式对话）
    auth 认证方式，aws-sdk 表示走 AWS SDK 凭证链（环境变量 → 共享配置 → 实例角色）
    models 模型列表，每个模型需要手动注册才能被 OpenClaw 识别

    每个模型条目：

    字段 说明
    id 模型 ID，对应 Bedrock 的推理配置 ID（如 us.amazon.nova-pro-v1:0）
    name 显示名称
    reasoning 是否支持推理/思考模式（DeepSeek R1 为 true）
    input 支持的输入类型，["text", "image"] 表示支持多模态
    contextWindow 上下文窗口大小（token 数）
    maxTokens 单次最大输出 token 数

    6.2 agents（Agent 配置）#

    {
  "agents": {
    "defaults": {
      "workspace": "C:\\Users\\colin\\.openclaw\\workspace",
      "model": {
        "primary": "amazon-bedrock/us.amazon.nova-pro-v1:0"
      }
    }
  }
}
    字段 说明
    workspace Agent 工作区路径，存放 AGENTS.md、SOUL.md 等文件
    model.primary 默认使用的模型，格式为 provider/model-id

    6.3 gateway（网关配置）#

    {
  "gateway": {
    "mode": "local",
    "auth": {
      "mode": "token",
      "token": "7f81f871..."
    },
    "port": 18789,
    "bind": "loopback",
    "tailscale": { "mode": "off" },
    "controlUi": { "allowInsecureAuth": true },
    "nodes": { "denyCommands": [...] }
  }
}
    字段 说明
    mode 运行模式，local 表示本地网关
    auth.mode 认证方式，token 表示用 Token 认证
    auth.token 自动生成的认证 Token，Web 控制台和客户端连接时使用
    port 监听端口，默认 18789
    bind 绑定地址，loopback 表示只监听 127.0.0.1（仅本机可访问，安全）
    tailscale Tailscale 远程访问，off 表示关闭
    controlUi.allowInsecureAuth 允许 Web 控制台非安全认证（本地使用无所谓）
    nodes.denyCommands 禁止的命令列表（摄像头、短信等敏感操作）

    6.4 session（会话配置）#

    {
  "session": {
    "dmScope": "per-channel-peer"
  }
}
    字段 说明
    dmScope DM 隔离策略。per-channel-peer 表示按频道+发送者隔离会话，多人使用时互不干扰

    6.5 tools（工具配置）#

    {
  "tools": {
    "profile": "coding"
  }
}
    字段 说明
    profile 工具集配置。可选值：minimal（最少工具）、coding（编程相关）、messaging（消息相关）、full（全部）

    注意： DeepSeek R1 不支持流式 Tool Use，如果用 R1 需要把 profile 改为 minimal 或想办法禁用工具。Nova Pro 支持所有 profile。

    6.6 plugins（插件配置）#

    {
  "plugins": {
    "entries": {
      "amazon-bedrock": {
        "config": {
          "discovery": {
            "enabled": true,
            "region": "us-east-1"
          }
        },
        "enabled": true
      }
    }
  }
}
    字段 说明
    entries.amazon-bedrock.enabled 是否启用 Bedrock 插件
    discovery.enabled 是否启用自动模型发现（扫描 AWS 账户中可用的 Bedrock 模型）
    discovery.region 发现模型的 AWS 区域

    6.7 skills（技能配置）#

    {
  "skills": {
    "entries": {
      "1password": { "enabled": false },
      "github": { "enabled": false },
      ...
    }
  }
}

    每个 skill 的 enabled: false 表示已禁用。通过 openclaw doctor 可以批量禁用不可用的 skills，减少系统提示词中的 skill 列表，节省 token。

    6.8 其他字段#

    字段 说明
    wizard 记录上次 onboarding/doctor 运行信息，无需手动修改
    meta 记录配置最后修改时间和版本，无需手动修改

    7. 常见问题#

    7.1 “openclaw” 命令找不到#

    运行以下命令查看全局包路径，确认该路径在系统 PATH 中：

    npm config get prefix

    7.2 “无法加载文件…因为在此系统上禁止运行脚本”#

    这是 PowerShell 执行策略限制。以管理员身份打开 PowerShell，执行：

    # 查看当前策略（默认可能是 Restricted）
Get-ExecutionPolicy

# 修改为 RemoteSigned（允许本地脚本运行，远程脚本需签名）
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

    验证修改生效后，重新运行 openclaw 命令即可。

    安全提示： 安装完成后如需恢复默认策略，可执行 Set-ExecutionPolicy Restricted -Scope CurrentUser。

    临时绕过（仅当前会话）： powershell -ExecutionPolicy Bypass -Command "你的命令"

    7.3 Gateway 服务安装失败（拒绝访问）#

    用管理员权限的 PowerShell 运行：

    openclaw gateway install

    如果仍然失败，OpenClaw 会自动回退到用户 Startup 文件夹方式实现开机自启。

    7.4 Anthropic 模型报地区限制错误#

    错误信息：Access to Anthropic models is not allowed from unsupported countries

    解决方案：换用 DeepSeek 或 Amazon Nova 系列模型，它们没有地区限制。

    7.5 plugins.allow 警告#

    出现 “plugins.allow is empty; discovered non-bundled plugins may auto-load” 警告时，配置明确允许的插件：

    openclaw config set plugins.allow '["amazon-bedrock"]'

    7.6 LiteLLM pricing fetch failed#

    无害警告，只是无法获取远程定价信息用于费用估算，不影响正常使用。

    7.7 AWS 凭证安全建议#

    • 不要将 Access Key 明文写入代码或文档
    • 定期轮换 Access Key
    • 使用最小权限原则，IAM 用户只需要 Bedrock 相关权限：
      • bedrock:InvokeModel
      • bedrock:InvokeModelWithResponseStream
      • bedrock:ListFoundationModels
      • bedrock:ListInferenceProfiles
