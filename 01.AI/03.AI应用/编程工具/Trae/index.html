    1.代码补全（根据注释自动补全）

    # 编写一个方法接受一个数组，并排序，排序使用冒泡排序，从小到大排序，并打印出数组中的所有元素
def bubble_sort(arr):
    for i in range(len(arr)):
        for j in range(len(arr) - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

arr = [1, 2, 3, 4, 5]
print(bubble_sort(arr)) #[1, 2, 3, 4, 5]

    智能体聊天模式（Agent Chat）#

    1.什么是智能体#

    工作流（Workflow）：把一个任务人为拆成固定步骤，按顺序执行的一种流程。
    智能体（Agent）：大模型处理任务的一种方法, 大模型动态拆解任务,去执行任务,可以调用工具、历史对话记忆,多轮推理,直到完成。

    Cursor、Trae 本质都是 Agent。

    2.TRAE 内置智能体与开发模式#

    TRAE 提供两种开发模式： IDE模式： 特点:保留传统开发流程，支持智能问答、代码补全等辅助功能，用户对开发过程有更强掌控感。 适用场景:适合需要精细控制代码或逐步验证逻辑的开发者。
    S0LO模式 特点:AI主导全流程(需求理解、编码、测试、部署)，通过自然语言输入即可自主完成开发任务，自动化程度高

    选择建议 1.若需保留开发自主性，选择IDE模式; 2如果你要追求效率或处理标准任务，SOLO模式更高效 快捷键切换模式：Ctrl + Alt + \ 打开智能体窗口：Ctrl+U

    3.IDE模式#

    IDE模式主要有三个智能体：

    • Chat
    • Builder
    • Builder with MCP

    1）Chat 最简单的模式。

    作用：

    聊天 + 写代码

    特点：

    • 可以理解代码
    • 可以生成代码
    • 不能直接创建文件

    例如：帮我写一个贪吃蛇游戏

    你现在是一名资深的网页游戏开发工程师，请帮我用原生HTML+CSS+JavaScript 编写一个可以直接在浏览器中运行的网页版《贪吃蛇》游戏，
要求如下 
功能要求： 
1.要求有积分统计 
2.页面要有多种背景可以切换 
3.代码添加中文注释

技术要求： 
不使用任何框架或库（如React、Vue、jQuery等） 
所有代码写在同一个HTML文件中 有详细注释，逻辑清晰 在浏览器中打开即可使用。
输出格式要求： 
一次性输出完整的HTML文件代码，并直接保存为snakeGame01.html文件到当前文件夹下 
代码可直接运行，无需任何依赖 包含<style>和<script>

    AI会输出代码：

    <html>
...
</html>

    但需要 手动复制保存。

    适合：

    • 代码学习
    • 代码解释
    • 代码片段生成

    2)Builder

    Builder 是 Chat 的升级版。

    区别：

    功能 Chat Builder
    写代码 ✔ ✔
    创建文件 ❌ ✔
    运行程序 ❌ ✔
    执行命令 ❌ ✔

    例如：

    生成一个贪吃蛇游戏
并保存为 snakeGame02.html

    Builder会自动：

    1 创建文件
2 写入代码
3 保存

    甚至可以执行：

    npm install
npm run dev

    3)Builder with MCP

    这是 最强的 Builder。

    因为它可以调用：

    MCP工具服务器

    MCP全称：

    Model Context Protocol

    可以理解为：

    AI的插件系统

    类似：

    类比 说明
    浏览器插件 扩展浏览器能力
    VSCode插件 扩展IDE能力
    MCP 扩展AI能力

    例如可以连接：

    • GitHub
    • 数据库
    • 浏览器
    • 本地文件
    • API服务

    AI就可以：

    读取数据库
调用API
控制浏览器
部署项目

    能力非常强。

    5.自定义智能体#

    自定义智能体:在Trae中创建自定义智能体可以帮你高效处理各种任务

    创建智能体的提示词

    • 名称:信息速览专家 自动搜索、整理信息并生成总结文档和可视化网页，适合快速了解新概念或行业报告
    • 名称:可视化讲师 将复杂概念转化为直观的可视化网页，通过图文并茂的方式帮你深入理解
    • 名称:知识库管理专家 知识库管理助手，专门管理本地文件，快速检索并理解文档内容，30秒内输出结构清晰的答案
    • 名称:MCP自动化助手 结合多种MCP工具，帮你完成文件操作、网页搜索、截图等重复性工作
    • 名称:Python技术专家 你是一名Python开发专家，非常擅长Python的各种框架，算法和性能优化
    • 名称:前端架构师 前端开发高级工程师，非常牛的那种，级别可以达到阿里P8

    应用实例1: 创建可视化讲师智能体，帮助学习编程技术

    使用可视化讲师智能体: 请用网页动画的形式，讲解一下冒泡排序

    应用实例2： 结合多种MCP工具，帮你完成文件操作、网页搜索、截图等重复性工作

    使用MCP自动化助手智能体: sales_data.xlsx根据商品分类，对每一个分类生成一个单独的文件，便于查看

    Editor 内 AI 编码（编辑器直控代码）#

    Trae上下文#

    在Trae工具里，“上下文(Context)“是指AI工具在回答你的问题或帮助你编程时所能”看到"和"理解"信息的范围，有助于Trae能精准响应。

    代码索引管理#

    对工作区中的代码进行全局索引构建，发起#引用上下文问答时将自动全局检索与问题相关的跨文件上下文，给出与项目更相关的回复。

    忽略文件#

    忽略文件配置的说明

    • 通过将.ignore’文件添加到根目录来控制哪些文件/文件夹被忽略。
    • 提升索引速度：排除依赖目录(比如:node_modules/、vendor/)、构建输出目录(比如:build/、out/)、大型媒体/数据文件(比如:.mp4、.mov、*.avi)
    • 避免干扰：某些文件与当前任务无关
