    提示词工程#

    提示词关键要素#

    一个高质量的提示词通常包含以下几个核心要素。 常见的记忆方式是 CRISPE 框架

    要素 英文 说明 示例
    角色与背景 Capacity & Role 为 AI 指定一个身份或使用场景，使回答更专业或更符合目标受众。 你是一位经验丰富的 Python 编程导师。
    任务与指令 Insight & Statement 清晰说明 AI 需要完成的任务，这是提示词最核心的部分。 请解释 Python 列表推导式，并给出三个由易到难的例子。
    步骤与约束 Procedure & Steps 将复杂任务拆解为步骤，或添加限制条件（长度、语气、风格等）。 按以下步骤回答：1. 一句话定义 2. 语法说明 3. 示例代码
    输出格式 Format & Output 指定结果的结构或格式，例如 Markdown、表格、JSON、代码块等。 请用表格展示方法、优点和缺点。
    输入示例 Examples 提供示例输入与输出，让 AI 模仿模式（Few-shot learning）。 如果输入“苹果”，输出“水果”；当输入“香蕉”时…

    CRISPE 其实解决的是三个问题,AI在回答前其实需要知道三件事

    问题 对应提示词
    你是谁？ Role
    我要做什么？ Task
    结果要长什么样？ Format

    其他都是 增强信息。

    常用的三种 Prompt 结构#

    1.简单任务（只需要 Task）

    解释什么是Python列表推导式

    2.常见结构（Task + Format）

    解释Python列表推导式，并用表格列出三个示例

    3.高质量结构（Role + Task + Format）

    你是一名Python编程导师。
请解释列表推导式，并给出三个由易到难的示例。
使用Markdown格式输出。

    常用 Prompt 模板#

    万能总模板#

    你现在是我的【资深程序员 + 架构师】  
语言：【Python / C++ / 其他】  
项目类型：【游戏辅助 / 自动化 / 工具脚本】  

当前状态：
- 项目是否已有代码：【是 / 否】
- 使用环境：【Windows / Linux】
- 目标平台：【x86 / x64】

目标：
【用一句话说明最终想干什么】

要求：
1. 在现有代码基础上修改（如果有）
2. 不要一次性写完，分步骤进行
3. 每一步都解释设计思路
4. 代码结构可扩展，方便后期维护
5. 优先使用稳定、常见方案

    从 0 到 1 建项目模板#

    我要从 0 写一个【项目名称】

功能需求：
1. 【功能 1】
2. 【功能 2】
3. 【功能 3】

技术要求：
- 语言：【Python / C++】
- 是否多线程：【是 / 否】
- 是否长期运行：【是 / 否】

请你：
1. 先设计整体架构（模块划分）
2. 再一步一步实现
3. 每一步写完等我确认

    Debug专用模板#

    下面是我当前的代码（不要重写）：

【粘贴代码】

期望行为：
【正常应该发生什么】

实际问题：
【现在发生了什么】

请你：
1. 先分析可能原因（不要直接改）
2. 指出具体出问题的代码位置
3. 给出修改方案
4. 再给我最终修改后的代码

    性能优化模板#

    这是当前代码：

【粘贴代码】

问题：
- 运行慢 / 卡顿 / CPU 占用高

请你：
1. 分析性能瓶颈
2. 说明是哪一部分导致的
3. 给出优化思路
4. 在不改变功能的前提下优化代码

    “只改一点点”模板（避免被重写）#

    在【文件名 / 函数名】中：

我要实现：
【新增 / 修改 的小功能】

限制：
1. 不要改其他逻辑
2. 不要重构
3. 保持原有接口不变

只给我：
- 需要修改的代码
- 修改原因说明

    重构（但不推翻）模板#

    这是当前项目结构：

【文件结构 + 说明】

问题：
- 代码越来越乱
- 耦合度高
- 不好加新功能

目标：
- 不改变功能
- 提高可维护性

请你：
1. 给出重构方案
2. 说明为什么这样拆
3. 一步一步重构

    学习型模板

    
我正在学习【技术点】

请你：
1. 用“给徒弟讲”的方式解释
2. 用最简单可运行示例
3. 告诉我常见坑
4. 给我练手任务

    创意与头脑风暴#

    我正在做 [项目描述]，遇到了 [问题/瓶颈]。
请帮我 brainstorm：
1. 可能的解决方案
2. 创新的实现思路
3. 可以借鉴的其他领域方案
4. 需要注意的风险

    录音转笔记

    请帮我整理录音内容整理为结构化的学习笔记，具体要求如下：

1. 整理原则：
内容精炼，避免冗余表达 - 不遗漏录音中的重点信息
不添加任何emoji或表情符号
确保结构清晰、重点突出、便于查阅。
对相似或重复内容进行合并归纳
保持原文重要数据和专业术语的准确性
如录音中存在明确的时间顺序或步骤，按顺序整理
如果录音有错误的部分，也请给我修正
输出为简体中文

2. 格式规范：
- 标题规范
标题层级可根据内容逻辑适当调整
## 1.xx介绍 / 2.xx安装 /3.xx使用（一级标题 → 二级标题）
### 各子章节（1.1二级标题 → 1.1.1三级标题）
#### / ##### / ###### 更深层级标题

- 涉及代码，命令部分必须包括
代码块用 ``` ``` 包裹

- 图片引用
图片引用格式为 ![图片](../images/图片.png)，兼容 Hugo 渲染

列表项使用
- 格式
