    AI 核心概念与术语梳理#

    人工智能（AI）是继互联网之后最伟大的技术变革。然而许多想拥抱 AI 的人，却常常陷入各种专业术语和流行词汇的泥沼中。本文旨在用通俗的语言，从底层到顶层系统梳理 AI 领域的核心概念，帮你建立清晰的知识框架。

    AI 的本质：四个核心问题#

    感知世界（看、听）→ 认知理解（读、写、说）→ 做出决策（推理、规划）→ 付诸行动（操控物理世界）

    AI 的所有技术方向，都在回答这四个问题中的一个或多个。理解这条主线，就能把纷繁的术语串成一条清晰的脉络。

    总览：四层结构#

    整个 AI 技术体系可以分为四层，从底到顶层层支撑：

    ┌──────────────────────────────────────────────────────────────────┐
│  第四层 · 应用层                                                  │
│  AI 产品 · API 聚合平台 · 开发框架 · 核心概念(Prompt/Agent/RAG)    │
├──────────────────────────────────────────────────────────────────┤
│  第三层 · 模型层                                                  │
│  基础模型 · LLM · 多模态模型 · 训练 · 微调 · 推理优化              │
├──────────────────────────────────────────────────────────────────┤
│  第二层 · 技术层                                                  │
│  NLP · 计算机视觉 · 语音技术 · Transformer · 推理技术              │
├──────────────────────────────────────────────────────────────────┤
│  第一层 · 基础层                                                  │
│  算力 · 算法(ML/DL) · 数据 · 基础学科                             │
└──────────────────────────────────────────────────────────────────┘
    应用层    你用的产品（ChatGPT、Cursor...）、平台（Bedrock、OpenRouter...）、框架（LangChain...）
  ↑ 调用
模型层    大模型（GPT-4、Claude...）+ 训练方法 + 推理优化
  ↑ 基于
技术层    NLP、CV、语音、Transformer 架构、推理技术
  ↑ 依赖
基础层    算力（GPU）+ 算法（ML/DL）+ 数据 + 基础学科（数学、统计...）

    理解这四层关系，就能建立起对整个 AI 领域的全局认知。每一层都在为上一层提供支撑，最终呈现为我们日常使用的 AI 产品和服务。

    第一层 · 基础层#

    AI 大厦的地基，提供算力、算法、数据和理论支撑。

    1.1 人工智能（AI, Artificial Intelligence）#

    让机器模拟人类智能的总称。只要机器能做出"看起来聪明"的事情，都算 AI。

    • 强 AI：具备人类级别的通用智能（目前还不存在）
    • 弱 AI / 窄 AI：只擅长特定任务，比如下棋、识别图片（目前所有 AI 都是这类）

    1.2 算力#

    为 AI 模型的训练和推理提供计算能力。训练大模型通常需要数千块 GPU 运行数周，成本极高。

    类别 内容
    核心硬件 GPU / TPU / NPU
    专用芯片 ASIC / FPGA / 神经网络芯片
    计算方式 并行计算、集群/云计算、边缘计算

    1.3 算法框架#

    用于构建和训练 AI 模型的软件工具。

    框架 背景
    PyTorch Meta 开源，学术界和工业界最流行
    TensorFlow Google 开源
    JAX Google 高性能计算框架
    MindSpore 华为开源
    PaddlePaddle 百度开源

    1.4 数据#

    AI 的"燃料"，模型的能力上限很大程度取决于数据质量。

    类别 说明
    结构化数据 数据库表格、CSV 等有固定格式的数据
    非结构化数据 文本、图片、音频、视频等
    多模态数据 同时包含多种类型的数据
    数据合成 用 AI 生成训练数据
    特征工程 从原始数据中提取有用特征
    数据清洗 去除噪声和错误数据
    数据标注 为数据添加标签，用于监督学习

    1.5 基础学科#

    AI 是一门高度交叉的学科，融合了多个领域的知识。

    学科 与 AI 的关系
    数学 线性代数、概率论、优化理论是 AI 的数学基础
    计算机科学 算法、数据结构、系统架构
    统计学 数据分析、模型评估
    语言学 NLP 的理论基础
    心理学 / 认知科学 理解人类智能，启发 AI 设计
    神经科学 / 生物学 神经网络的灵感来源
    哲学 / 逻辑学 AI 伦理、推理基础
    控制论 / 机器人学 自动控制、具身智能
    信息论 信息编码、数据压缩

    第二层 · 技术层#

    AI 的核心方法论和技术方向：机器怎么"学会"做事。

    AI 的五大方向（感知→认知→决策→行动）#

    感知世界（看、听）→ 认知理解（读写说）→ 做出决策（推理规划）→ 付诸行动（操控物理世界）
    方向 核心能力 成熟度 代表
    计算机视觉 感知（看） ⭐⭐⭐⭐ ViT、YOLO、Stable Diffusion
    语音技术 感知（听说） ⭐⭐⭐⭐ Whisper、VALL-E
    NLP / LLM 认知（读写说） ⭐⭐⭐⭐⭐ GPT-4、Claude
    强化学习 / Agent 决策（推理规划） ⭐⭐⭐ AlphaGo、AutoGPT
    具身智能 行动（动手做） ⭐⭐ 特斯拉 Optimus、自动驾驶

    融合趋势：多模态（⭐⭐⭐，快速发展中）：GPT-4o、Gemini 等模型正在把感知、认知、决策能力融合到一个模型里。AI 正在从"各方向各自发展"走向"统一的多模态基础模型"。

    语言、视觉、语音是 AI 的三大感知/认知方向，强化学习提供决策能力，具身智能让 AI 走进物理世界，多模态是融合趋势。

    2.1 机器学习（ML, Machine Learning）#

    AI 的一个子领域。核心思想：不直接编写规则，而是让机器从数据中自己学习规则。

    打个比方：

    • 传统编程 = 你告诉电脑"如果温度 > 30°就开空调"
    • 机器学习 = 你给电脑大量"温度 + 是否开空调"的数据，让它自己总结出规律

    学习方式：

    类型 说明 例子
    监督学习 给带标签的数据，让模型学习输入→输出的映射 垃圾邮件分类
    无监督学习 给不带标签的数据，让模型自己发现规律 用户分群
    强化学习 通过奖惩机制让模型学习最优策略 AlphaGo 下棋
    自监督学习 从数据本身构造标签来学习 GPT 预训练

    核心算法：

    算法 说明
    CNN 卷积神经网络，擅长图像处理
    RNN 循环神经网络，擅长序列数据
    LSTM 长短期记忆网络，RNN 的改进版
    GAN 生成对抗网络，用于生成内容
    VAE 变分自编码器，用于生成和压缩

    2.2 深度学习（DL, Deep Learning）#

    机器学习的一个子集，使用神经网络（模仿人脑神经元结构）来学习。“深度"指的是神经网络有很多层。

    • 层数越多，能学到的特征越复杂
    • 需要大量数据和算力
    • 是当前 AI 爆发的核心驱动力

    包含关系：

    人工智能 (AI)           ← 最大的概念
  └── 机器学习 (ML)      ← AI 的子领域
        └── 深度学习 (DL)  ← ML 的子集，用神经网络

    2.3 自然语言处理（NLP）#

    让机器理解和生成人类语言。

    方向 具体技术
    语言理解（NLU） 分词、词性标注、命名实体识别（NER）、情感分析
    语言生成（NLG） 文本生成、机器翻译、文本摘要
    文本表示 词嵌入（Word Embedding）、向量化、Seq2Seq
    文本挖掘 信息抽取、关系抽取、知识图谱

    2.4 计算机视觉（CV）#

    AI 的分支，使计算机能"看懂"并理解图像和视频内容。CNN 和 Transformer 架构推动其性能大幅提升，广泛应用于安防、自动驾驶、医疗影像等领域。

    CV 下的核心任务：

    计算机视觉 (CV)
  ├── 识别类任务
  │     ├── 图像分类 —— 整张图属于什么类别
  │     ├── 目标检测 —— 图中有什么、在哪里
  │     ├── 人脸识别 —— 识别人脸身份
  │     └── OCR —— 从图片中提取文字
  ├── 分割类任务
  │     ├── 语义分割 —— 每个像素属于什么类别
  │     ├── 实例分割 —— 区分同类的不同个体
  │     └── 全景分割 —— 语义 + 实例的结合
  ├── 视频类任务
  │     ├── 目标跟踪 —— 在视频中持续追踪目标
  │     └── 视频分析 —— 行为识别、异常检测等
  ├── 生成类任务
  │     ├── 图像生成 —— 创建全新图像
  │     └── 图像超分辨率 —— 低分辨率→高分辨率
  └── 架构
        └── ViT —— 将 Transformer 应用于视觉

    详细说明：

    任务 说明 代表方法/模型
    图像分类 识别整张图像所属的类别（如"猫"“狗"“汽车”）。准确率已在 ImageNet 等数据集上超越人类 ResNet、ViT
    目标检测 在图像中定位（画出边界框）并识别多个目标。两阶段方法精度高，单阶段方法速度快 Faster R-CNN（两阶段）、YOLO（单阶段）
    语义分割 为每个像素分配语义类别（如"道路"“天空"“行人”），不区分同类个体 U-Net、DeepLab
    实例分割 在检测目标的同时，为每个实例分割出精确的像素级掩膜，能区分同类的不同个体 Mask R-CNN
    全景分割 语义分割 + 实例分割的结合，既标注类别又区分个体 —
    人脸识别 基于人脸特征进行身份验证（1:1）或识别（1:N）。流程：检测→关键点定位→特征提取→比对 ArcFace
    OCR 将图像中的文字（印刷体、手写体）转换为可编辑文本。深度学习 OCR 能处理复杂背景和多语言 PaddleOCR、Tesseract
    目标跟踪 在视频序列中持续定位特定目标的运动轨迹，需克服遮挡、形变、光照变化等挑战 DeepSORT（多目标）
    视频分析 从视频流中提取时空信息，包括行为识别、异常事件检测、动作识别、视频摘要等 SlowFast
    图像生成 创建全新的逼真图像。GAN 通过对抗训练生成，扩散模型通过逐步去噪生成 StyleGAN（GAN）、Stable Diffusion / DALL·E（扩散模型）
    图像超分辨率 从低分辨率图像恢复高分辨率图像，补充细节信息 ESRGAN
    ViT（视觉 Transformer） 将图像分割成固定大小的图像块（Patch），用 Transformer 处理，在多项任务上超越传统 CNN ViT、DeiT

    图像识别是一个广义术语，涵盖分类、检测、分割等任务，指计算机对图像内容的整体理解能力。从早期的手工特征（SIFT、SURF）到深度学习自动特征学习，能力已大幅提升。

    2.5 语音技术#

    让机器听懂和生成语音。

    方向 说明
    语音识别（ASR） 语音转文字
    语音合成（TTS） 文字转语音
    语音克隆 复制特定人的声音
    声纹识别 通过声音识别身份
    情感识别 从语音中识别情绪
    语音翻译 实时语音翻译
    降噪 / 音频分离 去除噪音、分离声源
    音乐生成 AI 作曲

    2.6 Transformer 架构#

    当前 AI 革命的核心架构，几乎所有大模型都基于此。

    核心组件：

    组件 说明
    编码器（Encoder） 理解输入信息
    解码器（Decoder） 生成输出内容
    自注意力（Self-Attention） 让模型关注输入中最相关的部分
    位置编码（Positional Encoding） 让模型理解词语的顺序
    Tokenization 将文本切分为 Token

    架构演进：

    技术 说明
    MoE（混合专家） 多个"专家"网络分工协作，提升效率
    Flash Attention 优化注意力计算，降低显存占用
    Multi-Query Attention 减少注意力计算量
    Speculative Decoding 推测性解码，加速生成
    ViT（Vision Transformer） 将 Transformer 应用于图像
    DiT（Diffusion Transformer） 将 Transformer 用于扩散模型（图像/视频生成）
    Mamba / SSM 状态空间模型，Transformer 的潜在替代方案

    2.7 推理技术#

    提升模型推理能力的方法。

    技术 说明
    Chain of Thought（CoT） 思维链，让模型一步步推理
    Tree of Thoughts 思维树，探索多条推理路径
    Self-Consistency 多次推理取一致结果
    ReAct 推理 + 行动结合
    Reflection 让模型反思和修正自己的输出
    Few-Shot Learning 给几个示例让模型学习
    Zero-Shot Learning 不给示例直接推理
    In-Context Learning 利用上下文中的信息学习

    第三层 · 模型层#

    基于技术层构建出来的具体模型，以及围绕模型的训练、优化、部署。

    技术层回答"怎么做”（方法论），模型层回答"做出了什么”（具体产物）。例如：NLP 是技术方向，LLM 是这个方向上做出来的模型。

    3.1 基础模型（Foundation Model，又称"根基模型”）#

    用海量通用数据预训练出来的大规模模型，它本身不针对任何特定任务，但可以通过微调或 Prompt 适配到各种下游任务。

    可以把它理解为一个"毛坯房"——基本结构已经建好，你可以根据需要装修成卧室、办公室或商铺。

    核心特点：

    • 大规模预训练：用互联网级别的海量数据训练
    • 通用性强：一个模型能适配多种任务（翻译、摘要、问答、编程…）
    • 可迁移：通过微调就能适应特定领域，不需要从零训练

    基础模型的分类：

    基础模型 (Foundation Model)
  ├── 大语言模型 (LLM) —— 专注文本（GPT-4、Claude、Llama）
  ├── 视觉基础模型 —— 专注图像/视频（ViT、SAM、Stable Diffusion）
  ├── 语音基础模型 —— 专注语音/音频（Whisper、VALL-E、Suno）
  ├── 多模态基础模型 —— 同时处理多种数据（GPT-4o、Gemini）
  └── ...

    每个领域都遵循同样的模式：先有理解能力（听懂/看懂），再有生成能力（会说/会画）。而多模态模型的趋势是把这些能力融合到一个模型里。

    3.2 大语言模型（LLM, Large Language Model）#

    基础模型中专注于语言处理的那一类。“大"体现在参数量巨大（数十亿到数万亿）。

    参数（Parameters）：模型在训练过程中学到的数值，可以理解为模型的"知识量”。

    • GPT-3：1750 亿参数
    • Llama 3：80 亿 ~ 4050 亿参数

    Token：LLM 处理文本的基本单位。不完全等于"字"或"词"：

    • 英文中，1 个 token ≈ 0.75 个单词
    • 中文中，1 个汉字通常 = 1~2 个 token
    • Token 数量决定了模型能处理的文本长度和计费方式

    上下文窗口（Context Window）：模型一次能"看到"的最大 token 数量。

    • GPT-4 Turbo：128K tokens（约 10 万字）
    • Claude：200K tokens（约 15 万字）
    • 上下文窗口越大，模型能处理的信息越多

    Token 和上下文窗口是 LLM / 大模型特有的概念。传统机器学习模型（如决策树、SVM）并不使用 Token 机制。

    常见误区：LLM 能生成图片吗？

    严格来说不能。LLM 的核心能力是处理文本。但现在很多产品（如 ChatGPT）背后已经升级为多模态模型，集成了图像生成能力，所以给人"LLM 也能画图"的错觉。准确地说：

    • 纯 LLM（如早期 GPT-3）→ 只能处理文本
    • 多模态模型（如 GPT-4o、Gemini）→ 能同时处理文本、图像、音频等

    日常交流中人们常把多模态模型也叫"大模型"或"LLM"，但技术上它们已经超越了 LLM 的范畴。

    3.3 生成式 AI（GenAI, Generative AI）#

    生成式 AI 不是一个独立的模型类别，而是一种能力特征——只要模型能够生成新内容，就属于生成式 AI。

    基础模型
  ├── LLM（文本）           ✅ 能生成文本 → 属于生成式 AI
  ├── 图像生成模型           ✅ 能生成图片 → 属于生成式 AI
  ├── 图像分类模型           ❌ 只做判断，不生成新内容 → 不属于
  ├── 多模态模型             ✅ 通常能生成多种内容 → 属于生成式 AI
  └── ...

    生成式 AI 的应用领域：

    • 文本生成：ChatGPT、Claude
    • 图片生成：Midjourney、DALL·E、Stable Diffusion
    • 音频生成：Suno
    • 视频生成：Sora、可灵
    • 代码生成：GitHub Copilot、Kiro

    简单来说：LLM、视觉模型、多模态模型是按"处理什么数据"分类的；生成式 AI 是按"能不能生成新内容"来描述的能力。两者是不同维度的划分。

    3.5 开源模型 vs 闭源模型#

    类型 说明 代表
    开源 模型权重公开，可自行部署和修改 Llama、Mistral、Qwen
    闭源 只能通过 API 使用，不公开模型细节 GPT-4、Claude

    3.6 模型的生命周期#

    一个模型从"白纸"到"上线"的完整过程：

    预训练 → 监督微调(SFT) → 对齐训练(RLHF/DPO) → 部署 → 推理
    阶段 说明
    预训练（Pre-training） 用海量通用数据学习基本规律，成本最高
    监督微调（SFT） 用高质量问答数据训练模型遵循指令
    RLHF 基于人类反馈的强化学习，让输出更符合人类偏好
    DPO 直接偏好优化，RLHF 的简化替代方案
    PPO 近端策略优化，RLHF 中常用的强化学习算法
    RLAIF 用 AI 反馈替代人类反馈，降低标注成本
    推理（Inference） 模型训练好后，用它来回答问题、生成内容。你每次和 ChatGPT 对话，就是在做推理

    3.7 微调相关#

    概念 说明
    Fine-tuning 用特定领域数据进一步训练模型。如：通用模型 → 医学论文微调 → 医学专用模型
    LoRA 低秩适配，只训练少量参数就能微调，大幅降低成本
    QLoRA LoRA + 量化，进一步降低显存需求

    3.8 推理优化#

    让模型跑得更快、更省资源。

    技术 说明
    vLLM 高效 LLM 推理引擎
    TensorRT-LLM NVIDIA 推理加速框架
    量化（GPTQ / AWQ） 降低模型精度以减少显存占用和加速推理
    KV Cache 缓存已计算的键值对，避免重复计算

    3.9 其他关键能力#

    能力 说明
    安全对齐 确保模型输出安全、无害、符合价值观
    多模态融合 让模型同时处理文本、图像、音频等
    向量检索 将数据转为向量，实现语义级别的搜索（RAG 的基础）

    第四层 · 应用层#

    面向用户的产品、平台、开发工具和核心应用概念。

    4.1 核心应用概念#

    Prompt（提示词）#

    你给 AI 的输入指令。Prompt 的质量直接影响 AI 输出的质量。

    Prompt Engineering（提示词工程）#

    设计和优化 Prompt 的技巧，让 AI 给出更好的回答。常见技巧：

    • 零样本（Zero-shot）：直接提问，不给示例
    • 少样本（Few-shot）：给几个示例，让 AI 模仿
    • 思维链（Chain of Thought, CoT）：让 AI 一步步推理

    Temperature（温度）#

    控制 AI 输出随机性的参数：

    • 温度低（如 0.1）：输出更确定、更保守
    • 温度高（如 0.9）：输出更随机、更有创意

    幻觉（Hallucination）#

    AI 一本正经地胡说八道。模型生成的内容看起来很合理，但实际上是错误的或编造的。这是当前 LLM 的主要问题之一。

    RAG（检索增强生成, Retrieval-Augmented Generation）#

    让 AI 先从外部知识库中检索相关信息，再基于检索结果生成回答。这样可以：

    • 减少幻觉
    • 让 AI 使用最新的或私有的数据
    • 不需要重新训练模型

    Agent（智能体）#

    能够自主规划、使用工具、完成复杂任务的 AI 系统。不只是"问答"，而是能"做事"。

    比如：你说"帮我订一张明天去上海的机票"，Agent 会自己搜索航班、比较价格、完成预订。

    代表项目：

    • OpenClaw：2025 年底推出的开源 AI Agent 框架，GitHub 上最火的 AI 项目之一。运行在你自己的硬件上（本地电脑、服务器甚至树莓派），可以连接 WhatsApp、Telegram 等消息应用，自主执行文件管理、网页浏览、邮件处理等任务。核心特点是自托管、开源、本地运行，数据不经过第三方。
    • AutoGPT、MetaGPT：其他知名的 Agent 框架

    MCP（Model Context Protocol，模型上下文协议）#

    一种标准化协议，让 AI 模型能够连接和使用外部工具、数据源。可以理解为 AI 的"USB 接口"——统一的标准让不同工具都能即插即用。

    Function Call（函数调用）#

    让模型在对话中调用外部函数或 API 的能力。比如模型判断需要查天气，就自动调用天气 API 获取结果。MCP 是 Function Call 思想的标准化演进。

    Embedding（向量嵌入）#

    将文本转为数字向量（一串数字）的技术。转换后可以用数学方法计算文本之间的相似度，是 RAG 和语义搜索的基础。

    AIGC（AI 生成内容）#

    AI Generated Content 的缩写，泛指 AI 生成的所有内容，包括文本、图片、视频、音乐、代码等。

    4.2 API 聚合平台#

    你不需要分别对接每家模型厂商的 API，聚合平台帮你"一站式"调用多个模型。

    什么是 API 聚合平台？

    想象一下：你想用 GPT-4、Claude、Llama 三个模型，正常情况下需要分别注册三家的账号、学习三套不同的 API 接口、管理三份密钥。API 聚合平台就像一个"万能转接头"，提供统一的 API 接口，背后帮你对接各家模型。

    核心价值：

    • 统一接口：一套 API 调用多个模型，降低开发成本
    • 灵活切换：随时切换不同模型，不用改代码
    • 统一计费：一个账单管理所有模型的费用
    • 额外能力：负载均衡、访问控制、用量监控等

    主流平台：

    平台 说明
    AWS Bedrock 亚马逊云的 AI 模型服务平台。提供 Claude、Llama、Mistral、Stable Diffusion 等多家模型的统一调用。特点是深度集成 AWS 生态（IAM 权限、CloudWatch 监控、VPC 网络隔离等），适合企业级使用。名字来源于"基岩"（bedrock），寓意为 AI 应用提供坚实的基础。不需要管理服务器，按 Token 用量付费
    Azure OpenAI 微软云上的 OpenAI 模型服务，提供 GPT-4、DALL·E 等，集成 Azure 企业级安全和合规能力
    Google Vertex AI Google 云的 AI 平台，提供 Gemini 等模型，集成 Google 云生态
    OpenRouter 第三方聚合平台，支持数百个模型，统一 API 格式，按量付费，适合个人开发者
    litellm 开源聚合平台，适合自建

    它们在技术栈中的位置：

    你的应用 / 产品
  ↓ 调用
API 聚合平台（Bedrock / OpenRouter / Azure OpenAI...）  ← 在这里
  ↓ 转发
各家模型（GPT-4 / Claude / Llama...）

    简单来说：API 聚合平台是模型层和应用层之间的桥梁，让开发者不用直接对接每个模型厂商。

    4.5 边缘部署（Edge Deployment）#

    把模型部署在本地设备（手机、电脑）上运行，而不是云端。优点是隐私好、延迟低，缺点是设备算力有限。

    AI 安全与伦理（贯穿所有层）#

    安全不属于某一层，而是贯穿整个 AI 体系的横切关注点。

    概念 说明
    对齐（Alignment） 让 AI 的行为符合人类的意图和价值观。核心问题：如何确保 AI 做我们想让它做的事？
    RLHF 用人类反馈训练 AI，让输出更安全、更有用（详见 3.6）
    护栏（Guardrails） 对 AI 输出的限制和过滤机制，防止生成有害、不当或不准确的内容
    可解释性（Explainability） 理解 AI 为什么做出某个决策的能力。目前深度学习模型大多是"黑箱"

    当前热门趋势#

    趋势 说明
    多模态（Multimodal） 模型能同时处理文本、图片、音频、视频等多种类型的数据
    小模型（SLM） 参数量较小但在特定任务上表现优秀的模型，适合资源有限的场景
    AI Agent 从"对话"走向"行动"，AI 能自主完成复杂任务
    AI 编程 AI 辅助甚至自主编写代码，改变软件开发方式
    本地化部署 出于隐私和成本考虑，越来越多企业选择本地部署模型
