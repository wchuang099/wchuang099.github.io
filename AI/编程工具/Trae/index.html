    智能体聊天模式（Agent Chat）#

    1.什么是智能体

    工作流（Workflow）：把一个任务人为拆成固定步骤，按顺序执行的一种流程。 智能体（Agent）：大模型处理任务的一种方法, 大模型动态拆解任务,去执行任务,可以调用工具、历史对话记忆,多轮推理,直到完成。

    Cursor、Trae 本质都是 Agent。

    2.TRAE 内置智能体与开发模式

    TRAE 提供两种开发模式： IDE模式： 特点:保留传统开发流程，支持智能问答、代码补全等辅助功能，用户对开发过程有更强掌控感。 适用场景:适合需要精细控制代码或逐步验证逻辑的开发者。 S0LO模式 特点:AI主导全流程(需求理解、编码、测试、部署)，通过自然语言输入即可自主完成开发任务，自动化程度高

    选择建议 1.若需保留开发自主性，选择IDE模式; 2如果你要追求效率或处理标准任务，SOLO模式更高效 快捷键切换模式：Ctrl + Alt + \ 打开智能体窗口：Ctrl+U

    3.IDE模式 IDE模式主要有三个智能体：

    • Chat
    • Builder
    • Builder with MCP

    1）Chat 最简单的模式。

    作用：

    聊天 + 写代码

    特点：

    • 可以理解代码
    • 可以生成代码
    • 不能直接创建文件

    例如：帮我写一个贪吃蛇游戏

    你现在是一名资深的网页游戏开发工程师，请帮我用原生HTML+CSS+JavaScript 编写一个可以直接在浏览器中运行的网页版《贪吃蛇》游戏，
要求如下 
功能要求： 
1.要求有积分统计 
2.页面要有多种背景可以切换 
3.代码添加中文注释

技术要求： 
不使用任何框架或库（如React、Vue、jQuery等） 
所有代码写在同一个HTML文件中 有详细注释，逻辑清晰 在浏览器中打开即可使用。
输出格式要求： 
一次性输出完整的HTML文件代码，并直接保存为snakeGame01.html文件到当前文件夹下 
代码可直接运行，无需任何依赖 包含<style>和<script>

    AI会输出代码：

    <html>
...
</html>

    但需要 手动复制保存。

    适合：

    • 代码学习
    • 代码解释
    • 代码片段生成

    2)Builder

    Builder 是 Chat 的升级版。

    区别：

    功能 Chat Builder
    写代码 ✔ ✔
    创建文件 ❌ ✔
    运行程序 ❌ ✔
    执行命令 ❌ ✔

    例如：

    生成一个贪吃蛇游戏
并保存为 snakeGame02.html

    Builder会自动：

    1 创建文件
2 写入代码
3 保存

    甚至可以执行：

    npm install
npm run dev

    AI就可以：

    读取数据库

    能力非常强。

    提示词工程#

    提示词关键要素#

    一个高质量的提示词通常包含以下几个核心要素。 常见的记忆方式是 CRISPE 框架

    要素 英文 说明 示例
    角色与背景 Capacity & Role 为 AI 指定一个身份或使用场景，使回答更专业或更符合目标受众。 你是一位经验丰富的 Python 编程导师。
    任务与指令 Insight & Statement 清晰说明 AI 需要完成的任务，这是提示词最核心的部分。 请解释 Python 列表推导式，并给出三个由易到难的例子。
    步骤与约束 Procedure & Steps 将复杂任务拆解为步骤，或添加限制条件（长度、语气、风格等）。 按以下步骤回答：1. 一句话定义 2. 语法说明 3. 示例代码
    输出格式 Format & Output 指定结果的结构或格式，例如 Markdown、表格、JSON、代码块等。 请用表格展示方法、优点和缺点。
    输入示例 Examples 提供示例输入与输出，让 AI 模仿模式（Few-shot learning）。 如果输入“苹果”，输出“水果”；当输入“香蕉”时…

    CRISPE 其实解决的是三个问题,AI在回答前其实需要知道三件事

    问题 对应提示词
    你是谁？ Role
    我要做什么？ Task
    结果要长什么样？ Format

    其他都是 增强信息。

    常用的三种 Prompt 结构#

    1.简单任务（只需要 Task）

    解释什么是Python列表推导式

    2.常见结构（Task + Format）

    解释Python列表推导式，并用表格列出三个示例

    3.高质量结构（Role + Task + Format）

    你是一名Python编程导师。
请解释列表推导式，并给出三个由易到难的示例。
使用Markdown格式输出。

    常用 Prompt 模板#

    万能总模板#

    你现在是我的【资深程序员 + 架构师】  
语言：【Python / C++ / 其他】  
项目类型：【游戏辅助 / 自动化 / 工具脚本】  

当前状态：
- 项目是否已有代码：【是 / 否】
- 使用环境：【Windows / Linux】
- 目标平台：【x86 / x64】

目标：
【用一句话说明最终想干什么】

要求：
1. 在现有代码基础上修改（如果有）
2. 不要一次性写完，分步骤进行
3. 每一步都解释设计思路
4. 代码结构可扩展，方便后期维护
5. 优先使用稳定、常见方案

    从 0 到 1 建项目模板#

    我要从 0 写一个【项目名称】

功能需求：
1. 【功能 1】
2. 【功能 2】
3. 【功能 3】

技术要求：
- 语言：【Python / C++】
- 是否多线程：【是 / 否】
- 是否长期运行：【是 / 否】

请你：
1. 先设计整体架构（模块划分）
2. 再一步一步实现
3. 每一步写完等我确认

    Debug专用模板#

    下面是我当前的代码（不要重写）：

【粘贴代码】

期望行为：
【正常应该发生什么】

实际问题：
【现在发生了什么】

请你：
1. 先分析可能原因（不要直接改）
2. 指出具体出问题的代码位置
3. 给出修改方案
4. 再给我最终修改后的代码

    性能优化模板#

    这是当前代码：

【粘贴代码】

问题：
- 运行慢 / 卡顿 / CPU 占用高

请你：
1. 分析性能瓶颈
2. 说明是哪一部分导致的
3. 给出优化思路
4. 在不改变功能的前提下优化代码

    “只改一点点”模板（避免被重写）#

    在【文件名 / 函数名】中：

我要实现：
【新增 / 修改 的小功能】

限制：
1. 不要改其他逻辑
2. 不要重构
3. 保持原有接口不变

只给我：
- 需要修改的代码
- 修改原因说明

【文件结构 + 说明】

- 耦合度高

目标：
- 不改变功能
- 提高可维护性

请你：
1. 给出重构方案
2. 说明为什么这样拆
3. 一步一步重构
