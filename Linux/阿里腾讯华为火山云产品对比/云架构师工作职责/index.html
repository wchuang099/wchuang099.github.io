    云架构师（云售前&实施&售后全链路能力）#

    一、公司商业模式#

    🔥 公司本质在做什么？

    连接客户需求 + 云厂商能力 → 实现商业变现

    三大核心业务

    （1）云分销

    卖云资源（返点 / 差价）

    云厂商：

    • 阿里云
    • 腾讯云
    • 华为云
    • 火山引擎

    （2）MSP（托管服务）

    （3）AI业务：AI定制开发/大模型聚合平台

    二、商业逻辑（核心认知）#

    客户 vs 云厂商 vs 你

    客户：有问题
云厂商：有产品
你：负责匹配 + 落地

    本质

    客户不会用云 → 你帮他用
客户不会用AI → 你帮他落地

    三、售前+ 实施+售后全流程（核心SOP）#

    标准链路

    线索 → 商机 → 交流 → 方案 → 成交 → 实施 → 验证 → 闭环 → 售后

    四、售前阶段（成交前）#

    1️⃣ 线索阶段

    客户特点：

    • 问价格
    • 问配置
    • 问能不能做

    你要做：

    技术解答 + 配置建议 + 产品选型

    ❗ 边界

    不报价 → 只给销售建议

    2️⃣ 商机阶段（关键能力）

    商机分类

    1. AI类
2. 迁移类
3. 运维类
4. 普通上云

    分类能力 = 售前核心能力

    3️⃣ 客户沟通核心方法（四件套）

    1. 场景
2. 需求
3. 痛点
4. 目标

    为什么“目标”最重要？

    没有目标 → 无法验收 → 一定扯皮

    五、方案阶段（决定成交）#

    必须输出三样东西：

    1️⃣ 调研表（现状）

    客户现在有什么

    2️⃣ 对标表（目标）

    迁到云上用什么

    3️⃣ 技术方案

    六、绘制架构图能力#

    必须包含：

    1️⃣ 两端

    源环境 → 目标环境

    2️⃣ 结构

    用户 → SLB → ECS → DB

    3️⃣ 数据迁移链路

    ECS → SMC
数据库 → DTS
文件 → rsync

    4️⃣ 流量路径

    DNS切换

    核心原则

    如涉及业务改造，先做迁移，再优化

    七、实施阶段（交付过程）#

    迁移 + 配合 + 风险控制

    八、项目闭环#

    核心一句话

    做完 ≠ 结束
闭环才结束

    1️⃣ 验证阶段

    数据校验 + 业务验证

    责任划分

    你：数据
客户：业务

    2️⃣ 三大闭环动作

    （1）文档交付

    • 架构图
    • 方案
    • 操作记录

    （2）成本清理

    删除按量资源

    否则：

    持续扣费

    （3）权限清理

    回收账号 / 密码 / AK

    九、售后体系#

    🔥 售后核心三件事

    响应 + 解决 + 闭环

    1️⃣ 响应（最重要）

    原则：

    先回复，再解决

    标准话术：

    好的，我这边看一下，稍后给您回复

    🔥 核心认知

    1分钟响应 ＞ 1小时解决

    2️⃣ 解决问题路径

    1. 自己解决
2. 指导客户
3. 工单
4. 找人

    3️⃣ 输出方式

    文档 / 截图 / 远程 / 手把手

    4️⃣ 闭环

    解决 → 确认 → 结束

    十、主动服务#

    🔥 本质

    客户没说 → 你先发现

    例如：

    • 没备份
    • 没安全
    • 架构差

    👉 你要：

    提出优化建议

    十一、售后 = 二次销售入口#

    #

    典型场景：

    客户只买了服务器

    你发现：

    缺备份 / 安全 / CDN

    👉 你要：

    告诉销售 → 推动成交

    ❗ 边界

    你不卖 → 你推动

    十二、基础能力地图#

    七大核心模块（云厂商常见分类）

    类别 内容
    计算 ECS、CVM
    存储 OSS、NAS
    网络 VPC、SLB
    数据库 RDS、Redis
    中间件 Kafka、RabbitMQ
    安全 WAF、DDoS
    大数据 Elasticsearch 等

    ❗ 2. 不做承诺
