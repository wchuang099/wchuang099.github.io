    迁移流程总结(阿里云)#

    一、云迁移整体架构#

    云迁移涉及 七大核心模块（云厂商常见分类）

    类别 内容
    计算 ECS、CVM
    存储 OSS、NAS
    网络 VPC、SLB
    数据库 RDS、Redis
    中间件 Kafka、RabbitMQ
    安全 WAF、DDoS
    大数据 Elasticsearch 等

    迁移时通常需要逐个模块进行迁移。

    二、迁移需确认通用的问题点#

    在实施迁移前，需要与客户确认以下关键问题：

    1. 数据迁移时间#

    指：

    腾讯云 → 阿里云
数据复制时间

    例如：

    • 系统盘
    • 数据盘
    • 数据库

    需要给客户 大致时间预估

    例如：

    数据量：500GB
预计迁移时间：3~5小时

    2. 切换时间#

    即：

    业务切换到新环境的时间

    例如：

    DNS切换
负载均衡切换
应用地址切换

    通常需要：

    • 运维护航
    • 技术人员待命

    3. 停机窗口（空窗期）#

    必须确认客户是否允许停机：

    类型 说明
    有停机窗口 可以停机迁移
    无停机窗口 必须在线迁移

    例如：

    凌晨 2:00 ~ 4:00

    如果：

    空窗期 = 0

    意味着：

    必须不停机迁移

    难度会大很多。

    4.业务迁移风险点#

    迁移可能影响：

    项目 说明
    IP地址 内网IP会变化
    公网IP 云平台不同
    License 可能绑定IP
    MAC地址 部分软件绑定

    需要客户确认：

    是否有License绑定

    5.人员协调#

    迁移通常涉及：

    角色
    客户运维
    开发
    DBA
    云厂商架构师

    6.迁移职责边界#

    云厂商角色：

    数据迁移

    不负责：

    业务测试

    原因：

    客户业务逻辑只有客户自己清楚。

    客户需要：

    • 功能测试
    • 回归测试
    • 压力测试

    三、ECS服务器迁移#

    参考：

    • 阿里云服务器迁移中心（SMC）

    1 迁移工具#

    服务器迁移中心

    SMC（Server Migration Center）

    支持迁移来源：

    • 腾讯云 CVM
    • AWS EC2
    • VMware
    • 物理服务器

    2 迁移方式#

    方案一（推荐）

    SMC在线迁移

    流程：

    腾讯云CVM
   ↓
安装 SMC Agent
   ↓
生成阿里云镜像
   ↓
创建 ECS 实例

    特点：

    • 支持在线迁移
    • 短暂停机
    • 自动生成镜像

    方案二

    镜像迁移

    流程：

    腾讯云制作镜像
   ↓
下载镜像
   ↓
导入阿里云
   ↓
创建ECS

    3 使用限制#

    • 只支持 VPC网络
    • 不支持跨架构迁移

    例如：

    x86 → x86
ARM → ARM

    不能：

    x86 → ARM

    4 CPU架构类型#

    需要注意CPU架构。

    阿里云常见CPU：

    类型 说明
    Intel 主流
    AMD 部分实例
    倚天Yitian 阿里云自研CPU

    倚天CPU：

    • 阿里云自研
    • ARM架构
    • 属于国产信创CPU

    5 网络迁移方式#

    迁移网络通常有三种：

    1 公网迁移

    腾讯云
 ↓
公网
 ↓
阿里云

    特点：

    • 简单
    • 速度受公网带宽限制

    2 VPN迁移

    通过 IPSec VPN

    腾讯云
 ↓
VPN
 ↓
阿里云

    特点：

    • 加密
    • 仍然走公网
    • 迁移速度受限于带宽

    3 专线迁移

    腾讯云机房
   ↓
专线
   ↓
阿里云机房

    特点：

    • 内网
    • 速度快

    但存在：

    • 施工周期
    • 端口费用
    • 流量费用

    四、数据库迁移#

    数据库迁移是 最关键的部分。

    因为：

    服务器可以重建 数据库不能丢

    1 数据库类型#

    迁移前必须先确认数据库类型。

    数据库分为两大类：

    关系型数据库

    常见：

    类型
    MySQL
    PostgreSQL
    SQL Server
    Oracle

    非关系型数据库

    类型
    Redis
    MongoDB

    2 数据库架构调研#

    需要调研：

    项目 内容
    数据库类型 MySQL / Redis
    是否PaaS RDS 或 自建
    架构 单机 / 主从
    数据量 100GB / 1TB
    版本 MySQL5.7 / 8.0
    网络 是否公网
    权限 是否允许迁移

    3 数据库迁移工具#

    阿里云：

    DTS（Data Transmission Service）

    功能：

    • 结构迁移
    • 全量迁移
    • 增量同步
    • 双向同步

    4 DTS费用#

    费用主要两部分：

    DTS实例费用

    根据规格收费。

    例如：

    规格 读取能力
    小 2000 rows/s
    大 10000 rows/s

    规格越高：

    • 迁移越快
    • 费用越高

    增量同步费用

    规则：

    类型 是否收费
    全量迁移 不收费
    增量同步 收费

    5 DTS风险#

    读取速度越快：

    对源数据库压力越大

    可能导致：

    数据库性能下降

    需要和客户确认：

    • 当前数据库负载
    • CPU利用率

    6 数据校验#

    迁移完成后需要校验：

    • 表数量
    • 行数
    • 数据一致性

    DTS提供自动校验功能。

    五、对象存储迁移#

    对象存储主要存储非结构化数据/静态资源

    腾讯云：

    COS

    阿里云：

    OSS

    1 OSS Import#

    支持：

    • 批量迁移
    • 自动校验
    • HTTP源

    流程：

    COS
 ↓
HTTP URL
 ↓
OSS Import
 ↓
OSS

    2 ossutil#

    适合大规模迁移

    示例：

    ossutil cp -r cos://bucket oss://bucket

    腾讯云：

    • CFS
    • GooseFS

    阿里云：

    • NAS
    • CPFS

    迁移方式：

    rsync

    示例：

    rsync -avz

    腾讯云：

    • CKafka
    • RabbitMQ

    阿里云：

    • Kafka
    • RabbitMQ

    1.Kafka迁移#

    使用

    MirrorMaker

    流程

    腾讯云Kafka
 ↓
MirrorMaker
 ↓
阿里云Kafka

    2.RabbitMQ迁移#

    推荐方案：

    双写 + 双读

    迁移步骤：

    1 元数据迁移 2 双读验证 3 双写双读 4 单写新集群 5 下线旧集群

    八、Elasticsearch迁移#

    1.快照迁移#

    腾讯云 ES
   ↓ snapshot
OSS

    支持：

    全量 + 增量

    适合实时迁移。
