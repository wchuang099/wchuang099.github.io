    1. Amazon 资源名称（ARN）#

    1.1 ARN 格式#

    arn:partition:service:region:account-id:resource-id
arn:partition:service:region:account-id:resource-type/resource-id
arn:partition:service:region:account-id:resource-type:resource-id

    字段说明：

    字段 含义
    partition aws（标准区域）、aws-cn（中国）、aws-us-gov（GovCloud）
    service AWS 服务命名空间，如 iam、s3、ec2
    region 区域代码，如 us-east-2；IAM 全局资源留空
    account-id 12 位账户 ID，无连字符
    resource-id 资源名称、ID、路径或复合标识

    示例：

    arn:aws:iam::123456789012:user/john
arn:aws:sns:us-east-1:123456789012:example-sns-topic-name
arn:aws:ec2:us-east-1:123456789012:vpc/vpc-0e9801d129EXAMPLE

    1.2 路径和通配符#

    • IAM 用户、组、角色、策略支持路径：/division_abc/subdivision_xyz/
    • 路径字符：字母数字 + / + = , . @ _ -
    • Resource / NotResource 元素可使用 * 和 ?
    • Principal / NotPrincipal 中不能使用通配符

    示例：

    arn:aws:iam::123456789012:role/*
arn:aws:iam::123456789012:role/service-*
arn:aws:s3:::amzn-s3-demo-bucket/Development/*
arn:aws:s3:::amzn-s3-demo-bucket/????-test

    1.3 通配符限制#

    • 资源类型部分不能使用通配符：arn:aws:lambda:...:functi*:... 非法
    • 分区、前缀（arn:aws）部分不能使用通配符
    • 基于身份策略的不完整 ARN 会自动补 *
    • 会话策略中不支持不完整 ARN，会触发 MalformedPolicyDocumentException

    2. IAM 标识符#

    2.1 易记名称#

    用户、组、角色、策略、服务器证书的可读标识。API 和 CLI 可添加可选路径，控制台无法创建或操作路径。

    2.2 IAM ARN 语法#

    arn:aws:iam::<account>:root
arn:aws:iam::<account>:user/<user-name-with-path>
arn:aws:iam::<account>:group/<group-name-with-path>
arn:aws:iam::<account>:role/<role-name-with-path>
arn:aws:iam::<account>:policy/<policy-name-with-path>
arn:aws:iam::<account>:instance-profile/<name-with-path>
arn:aws:sts::<account>:federated-user/<user-name>
arn:aws:sts::<account>:assumed-role/<role-name>/<session-name>
arn:aws:sts::<account>:self
arn:aws:iam::<account>:mfa/<device-name-with-path>
arn:aws:iam::<account>:u2f/<token-id>
arn:aws:iam::<account>:server-certificate/<cert-name-with-path>
arn:aws:iam::<account>:saml-provider/<provider-name>
arn:aws:iam::<account>:oidc-provider/<provider-name>
arn:aws:iam::aws:contextProvider/<name>

    2.3 唯一标识符#

    每个 IAM 资源有一个不可变的唯一 ID（前缀 + 随机字符串）：

    前缀 资源
    AIDA IAM 用户
    AGPA 用户组
    AROA 角色
    AIPA EC2 实例配置文件
    ANPA 托管策略
    ANVA 托管策略版本
    AKIA 访问密钥
    ASIA STS 临时访问密钥 ID
    ABIA STS 服务持有者令牌
    APKA 公有密钥
    ASCA 证书
    ACCA 上下文特定凭证

    使用场景：易记名称删除后重复使用时，策略基于唯一 ID 可避免新旧用户继承旧权限。

    "Principal": {
  "AWS": [
    "arn:aws:iam::111122223333:role/role-name",
    "AIDACKCEVSQ6C2EXAMPLE",
    "AROADBQP57FF2AEXAMPLE"
  ]
}

    或使用 aws:userId 条件键：

    "Condition": {
  "StringLike": {
    "aws:userId": [
      "AIDACKCEVSQ6C2EXAMPLE",
      "AROADBQP57FF2AEXAMPLE:session-name",
      "AROA1234567890EXAMPLE:*",
      "111122223333"
    ]
  }
}

    2.4 获取唯一 ID#

    CLI：

    aws sts get-caller-identity
aws iam get-user --user-name <n>
aws iam get-role --role-name <n>
aws iam get-group --group-name <n>
aws iam get-policy --policy-arn <arn>
aws iam get-instance-profile --instance-profile-name <n>
aws iam get-server-certificate --server-certificate-name <n>

    3. IAM 和 AWS STS 配额#

    3.1 账户级主要配额（部分）#

    实体 默认上限
    IAM 用户 5000
    IAM 组 300
    IAM 角色 1000（可申请提升）
    客户管理型策略 1500
    策略版本 5
    实例配置文件 1000
    每个 IAM 用户附加托管策略 10
    每个 IAM 角色附加托管策略 10
    每个用户属于的组 10
    每个组包含的用户 无硬上限（受组数限制）
    SAML 提供者 100
    OIDC 提供者 100
    服务器证书 20

    3.2 大小限制#

    对象 最大字符数
    用户名 / 角色名 / 组名 / 策略名 64
    路径 512
    用户描述 1000
    托管策略文档 6144
    IAM 用户内联策略总和 2048
    IAM 组内联策略总和 5120
    IAM 角色内联策略总和 10240
    会话策略大小 2048（传递前 JSON 打包前）
    PackedPolicySize 上限 10240（打包后）

    3.3 AWS STS 配额#

    • AssumeRole*：每秒 2000 请求（默认）
    • GetCallerIdentity：每秒 10,000
    • 临时凭证最长 12 小时（取决于角色最大会话时长）；角色链最多 1 小时
    • 每账户 MFA 设备上限：每用户 8 台，根用户 8 台

    4. 双堆栈与 IPv6 端点#

    IAM 提供 IPv4/IPv6 双堆栈端点：

    • 公有：iam.amazonaws.com 全局；iam-fips.amazonaws.com FIPS
    • 双堆栈：iam.global.api.aws、iam-fips.global.api.aws
    • STS 区域端点：sts.<region>.api.aws（双堆栈）、sts.<region>.amazonaws.com（标准）

    应用可将 SDK 配置为优先使用双堆栈端点以支持 IPv6 客户端。

    5. 接口 VPC 端点#

    IAM 通过 AWS PrivateLink 支持接口 VPC 端点：

    • IAM VPC 端点：com.amazonaws.<region>.iam（仅 us-east-1 等特定区域支持全局端点）
    • STS VPC 端点：com.amazonaws.<region>.sts
    • 可通过端点策略进一步限制可用操作
    • 仅以下区域支持 IAM VPC 端点：列表见官方文档，随时间扩展

    6. AWS 签名版本 4#

    API 请求使用 SigV4 进行签名：

    1. 计算规范请求字符串（HTTPMethod、URI、QueryString、Headers、Payload hash）
    2. 构造待签字符串（AWS4-HMAC-SHA256 + timestamp + 凭证域 + 规范请求哈希）
    3. 使用 SigningKey = HMAC(HMAC(HMAC(HMAC(k=kSecret, date), region), service), "aws4_request")
    4. 签名 = HEX(HMAC(SigningKey, StringToSign))

    放入 Authorization 头：

    Authorization: AWS4-HMAC-SHA256 Credential=AKIDEXAMPLE/20260507/us-east-1/iam/aws4_request,
    SignedHeaders=host;x-amz-date,
    Signature=calculatedSignature

    6.1 SigV4a#

    SigV4a 是 SigV4 的扩展版，用于多区域场景（如 S3 多区域接入点）：

    • 使用非对称密钥生成签名
    • 可跨区域验证，无需每区域独立签名
    • AWS SDK 自动支持

    6.2 使用建议#

    • 优先让 SDK 处理签名
    • 手动实现时使用 AWS 提供的 SigV4 测试套件验证
    • 所有请求使用 HTTPS（aws:SecureTransport = true）

    7. JSON 策略语言参考#

    7.1 顶层元素#

    元素 说明
    Version 2012-10-17（推荐），2008-10-17 为旧版
    Id 可选策略 ID
    Statement 语句数组

    7.2 语句元素#

    元素 说明
    Sid 语句标识（同一策略唯一）
    Effect Allow 或 Deny
    Principal / NotPrincipal 基于资源的策略中指定主体
    Action / NotAction 操作或操作数组
    Resource / NotResource 资源 ARN
    Condition 条件映射

    7.3 Principal 形式#

    "Principal": { "AWS": "arn:aws:iam::111:root" }
"Principal": { "AWS": ["arn:...:user/A", "arn:...:role/B"] }
"Principal": { "Federated": "arn:...:saml-provider/X" }
"Principal": { "Service": "ec2.amazonaws.com" }
"Principal": { "CanonicalUser": "<canonical-id>" }
"Principal": "*"  // 任意主体，需配合条件

    7.4 数据类型#

    条件值的 JSON 类型：

    • 字符串、数字、布尔、日期时间、IP 地址、ARN、Null、Binary
    • 多值通过数组表示
    • 单值键：只允许一个值；多值键可传多个值

    7.5 多值键与集合运算符#

    • ForAllValues:<op>：请求键的所有值都必须匹配列表
    • ForAnyValue:<op>：至少一个值匹配
    • 多值键示例：aws:TagKeys

    7.6 重要全局条件键（补充）#

    条件键 用途
    aws:PrincipalOrgID 主体所在组织 ID
    aws:PrincipalOrgPaths 主体所在组织路径
    aws:PrincipalIsAWSService 主体是否为 AWS 服务
    aws:PrincipalServiceName 服务名
    aws:PrincipalServiceNamesList 服务名列表
    aws:PrincipalAccount 主体 AWS 账户
    aws:UserAgent 请求的 User-Agent
    aws:VpcSourceIp VPC 内源 IP
    aws:SourceVpc 源 VPC
    aws:SourceVpce 源 VPC 端点
    aws:EpochTime Unix 时间戳
    aws:RequestedRegion 请求的目标区域

    7.7 策略变量语法#

    ${aws:username}
${aws:userid}
${aws:PrincipalTag/TagKey}
${aws:SourceIp}
${saml:sub}
${token.actions.githubusercontent.com:sub}

    8. 使用 IAM 的 AWS 服务#

    所有支持 IAM 的服务（超过 200 个）按能力分类：

    能力 说明
    Actions 支持在策略中指定的操作列表
    Resource-level 支持针对具体资源 ARN 授权
    Resource-based policies 资源可附加策略
    ACLs 支持访问控制列表
    ABAC 支持标签条件授权
    Temporary credentials 接受 STS 临时凭证
    Service-linked roles 支持服务相关角色
    Service roles 支持服务角色

    完整列表参见官方文档的“使用 IAM 的 AWS 服务”。

    9. 简化 API 命名空间与别名#

    部分服务为了兼容，在 IAM 策略中存在多种命名空间（如 logs 与 cloudwatch-logs）。建议：

    • 查阅每项服务的操作、资源和条件键参考（SAR）
    • 使用官方策略生成工具避免名称拼写错误
    • 避免在生产环境混用不同命名空间

    10. IAM 与其他服务协作#

    10.1 CloudFormation#

    • AWS::IAM::User / ::Group / ::Role / ::Policy / ::InstanceProfile / ::ManagedPolicy / ::AccessKey
    • 可通过 Transform 和宏简化复杂策略编写
    • 模板中的用户、组、角色、策略都支持 Path 属性

    10.2 AWS CloudShell#

    • 浏览器内终端，使用当前控制台登录身份的权限
    • 免费每月 1 GB 存储
    • 要求 AWSCloudShellFullAccess 托管策略或等效权限

    10.3 AWS SDK 与 CLI#

    SDK 凭证提供链（优先级从高到低）：

    1. 显式传入的凭证
    2. 环境变量（AWS_ACCESS_KEY_ID、AWS_SECRET_ACCESS_KEY、AWS_SESSION_TOKEN）
    3. 本地 AWS 共享配置（~/.aws/credentials）
    4. 本地 AWS 配置文件（~/.aws/config）
    5. IAM Identity Center SSO 会话
    6. EC2 实例元数据 / ECS 任务角色 / EKS IRSA

    11. HTTP Query API#

    IAM 支持直接 HTTPS Query API，用于不便使用 SDK 的场景。

    11.1 端点#

    • 全局：https://iam.amazonaws.com/
    • 中国：https://iam.cn-north-1.amazonaws.com.cn/
    • GovCloud：https://iam.us-gov.amazonaws.com/

    STS：支持区域端点 https://sts.<region>.amazonaws.com。

    11.2 要求#

    • 必须使用 HTTPS
    • 所有请求需使用 SigV4 签名
    • 建议使用 SDK 而非手动构造请求
