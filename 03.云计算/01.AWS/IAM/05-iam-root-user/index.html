    1. AWS 账户根用户概览#

    创建 AWS 账户时自动生成的唯一身份，拥有账户中所有服务和资源的完全访问权限。登录凭证是创建账户时使用的邮箱和密码。

    核心原则：

    • 仅用于执行必须根凭证的任务
    • 默认强制启用 MFA（客户需手动添加）
    • 日常任务使用 IAM Identity Center、IAM 用户或角色

    2. 需要根用户凭证的任务#

    2.1 账户管理任务#

    • 更改 AWS 账户设置（独立账户）：邮箱、根用户密码、根用户访问密钥
    • 关闭 AWS 账户（独立账户）
    • 恢复 IAM 用户权限：当唯一 IAM 管理员撤销了自己权限时通过根用户恢复

    对于 AWS Organizations，管理账户或委托管理员可以集中管理成员账户：关闭成员账户、更新根邮箱、账户名、联系人、备用联系人、AWS 区域。

    2.2 账单与税务任务#

    • 激活 IAM 对账单和成本管理控制台的访问权限
    • 某些账单任务仅限根用户
    • 查看特定税务发票（如 AWS 欧洲 VAT 发票；AWS Inc 或 AISPL 的 VAT 发票需 IAM 用户有 aws-portal:ViewBilling）

    2.3 AWS GovCloud (US)#

    • 注册 AWS GovCloud (US)
    • 向支持请求 GovCloud 根用户访问密钥

    2.4 其他服务#

    服务 仅限根用户的任务
    Amazon EC2 在预留实例 Marketplace 注册为卖家
    AWS KMS 密钥变得不可管理时通过 Support 恢复
    Amazon Mechanical Turk 关联 AWS 账户到 mTurk 请求者账户
    Amazon S3 为存储桶启用 MFA；编辑或删除拒绝所有主体的桶策略

    2.5 集中根访问下的特权任务#

    启用 AWS Organizations 集中根访问后，成员账户可以通过短期特权会话执行以下原需根凭证的任务：

    • 删除成员账户根用户凭证
    • 允许密码恢复（恢复成员账户根凭证）
    • 删除配置错误的 S3 存储桶策略
    • 删除配置错误的 SQS 队列策略

    3. 集中根访问管理（AWS Organizations）#

    3.1 核心能力#

    • 从成员账户中移除根凭证（密码、访问密钥、签名证书、MFA）
    • 新建成员账户默认无根凭证
    • 成员账户不能登录根用户或执行密码恢复（除非启用账户恢复）
    • 管理账户或委托管理员可代表成员账户执行特定特权任务

    3.2 前置 IAM 权限#

    启用功能需要的权限：

    • iam:EnableOrganizationsRootCredentialsManagement
    • iam:EnableOrganizationsRootSessions
    • iam:ListOrganizationsFeatures
    • organizations:EnableAwsServiceAccess
    • organizations:ListAccountsForParent
    • organizations:RegisterDelegatedAdministrator

    审计成员账户根凭证状态建议的权限：

    • iam:GetAccessKeyLastUsed
    • iam:GetAccountSummary
    • iam:GetLoginProfile
    • iam:GetUser
    • iam:ListAccessKeys
    • iam:ListMFADevices
    • iam:ListSigningCertificates
    • sts:AssumeRoot

    可使用 IAMAuditRootUserCredentials 托管策略缩小审计权限范围。

    3.3 启用集中根访问#

    控制台：IAM → 根访问权限管理 → 启用 → 选择要启用的功能：

    • 根凭证管理：允许删除成员账户根凭证
    • 成员账户中的特权根操作：允许执行特定根用户任务

    可选：指定委托管理员账户 ID。

    CLI：

    # 启用可信访问
aws organizations enable-aws-service-access \
    --service-principal iam.amazonaws.com

# 启用根凭证管理
aws iam enable-organizations-root-credentials-management

# 启用特权根会话
aws iam enable-organizations-root-sessions

# 注册委托管理员
aws organizations register-delegated-administrator \
    --service-principal iam.amazonaws.com \
    --account-id 111111111111

    API：EnableAWSServiceAccess、EnableOrganizationsRootCredentialsManagement、EnableOrganizationsRootSessions、RegisterDelegatedAdministrator。

    3.4 执行特权任务#

    启动特权会话需使用 sts:AssumeRoot，不支持全局端点，必须向区域 STS 端点发送请求。

    可选的任务范围策略（AWS 托管）：

    • IAMAuditRootUserCredentials：审计根凭证状态
    • IAMCreateRootUserPassword：创建根密码
    • IAMDeleteRootUserCredentials：删除根凭证
    • S3UnlockBucketPolicy：解锁 S3 桶策略
    • SQSUnlockQueuePolicy：解锁 SQS 队列策略

    使用 STS 条件键 sts:TaskPolicyArn 可限制可担任的任务策略。

    CLI 示例（委托管理员删除成员账户根凭证）：

    aws sts assume-root \
    --target-principal 111122223333 \
    --task-policy-arn arn=arn:aws:iam::aws:policy/root-task/IAMDeleteRootUserCredentials \
    --duration-seconds 900

    使用返回的临时凭证执行操作：

    操作类别 CLI 命令
    检查根凭证状态 get-user、get-login-profile、list-access-keys、list-signing-certificates、list-mfa-devices、get-access-key-last-used
    删除根凭证 delete-login-profile、delete-access-key、delete-signing-certificate、deactivate-mfa-device
    修复 S3 桶策略 list-buckets、get-bucket-policy、put-bucket-policy、delete-bucket-policy
    修复 SQS 队列策略 list-queues、get-queue-url、get-queue-attributes、set-queue-attributes
    允许密码恢复 get-login-profile、create-login-profile

    3.5 控制台特权操作流程#

    1. 导航到 IAM → 根访问管理
    2. 从成员账户列表选择目标账户 → 采取特权操作
    3. 选择操作：
      • 删除 S3 存储桶策略：浏览桶 → 删除策略，随后在 S3 控制台重建
      • 删除 SQS 策略：输入队列名 → 删除
      • 删除根凭证：移除密码、访问密钥、签名证书、MFA
      • 允许密码恢复（仅当成员账户无根凭证时可用）：发件箱接收者可以重置密码

    4. 根用户的 MFA 要求#

    • 所有账户类型（独立、管理、成员）根用户必须配置 MFA
    • 未启用 MFA 时，首次登录后 35 天内必须注册
    • 支持最多 8 个 MFA 设备任意组合
    • 推荐抗钓鱼设备：FIDO 安全密钥、通行密钥
    • 其他支持：TOTP 硬件令牌、虚拟 MFA 应用
