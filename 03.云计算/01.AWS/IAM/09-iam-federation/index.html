    1. 身份提供商与联合概述#

    联合身份验证让外部 IdP 管理用户身份，外部用户通过角色获得 AWS 临时凭证，避免在应用中嵌入长期密钥。

    1.1 联合类型对比#

    联合类型 适用账户 适合的身份 支持的身份源
    使用 IAM Identity Center 的联合 AWS Organizations 多账户 员工人类用户 SAML 2.0、Managed AD、Identity Center 目录
    使用 IAM 的联合 单一独立账户 小规模人类用户、机器用户 SAML 2.0、OIDC
    Amazon Cognito 身份池 任意 应用程序用户 SAML 2.0、OIDC、OAuth 2.0 社交 IdP

    推荐：多账户组织使用 IAM Identity Center，应用用户使用 Cognito，单账户短期场景使用 IAM SAML/OIDC。

    1.2 工作负载联合#

    • IAM Roles Anywhere：AWS 外部工作负载通过 X.509 证书获取临时凭证
    • 外部 IdP (SAML/OIDC)：通过 AssumeRoleWithSAML / AssumeRoleWithWebIdentity
    • AWS IoT Core：物联网设备通过 MTLS

    2. OIDC 联合#

    2.1 典型流程#

    移动应用使用外部 IdP（Login with Amazon、Facebook、Google、其他 OIDC）：

    1. 应用调用 IdP 登录接口
    2. IdP 验证用户并返回 JWT
    3. 应用调用 AssumeRoleWithWebIdentity（或经 Cognito 身份池的 GetCredentialsForIdentity）
    4. AWS STS 返回临时凭证
    5. 应用使用凭证访问 AWS 资源

    临时凭证默认 1 小时，过期前需重新获取。

    2.2 Amazon Cognito 路径#

    首选方案：使用 Cognito 身份池作为凭证代理，配合：

    • 用户池（登录、注册、个人资料）
    • 身份池（为 IAM 角色颁发临时凭证）

    集成步骤：

    1. 开发者注册 IdP 并配置应用
    2. 在 Cognito 控制台创建身份池（自动创建两套角色：已验证、未验证 Guest）
    3. 集成 AWS Amplify 到应用
    4. 创建 Cognito 凭证提供程序实例，传入身份池 ID、账户号、角色 ARN
    5. 应用使用该凭证提供程序访问 AWS 客户端

    Google、Facebook、Amazon Cognito 内置为 AWS 的 OIDC 提供商，无需额外创建 IAM OIDC 身份提供商。

    2.3 IAM OIDC 身份提供者#

    2.3.1 先决条件#

    IdP 必须具备 OIDC 发现文档（/.well-known/openid-configuration），包含：

    • issuer：域 URL（https:// 开头，不含端口）
    • jwks_uri：JWKS 端点，最多 100 个 RSA 密钥和 100 个 EC 密钥
    • claims_supported：至少 aud、iat、iss、sub
    • response_types_supported：id_token
    • subject_types_supported：public
    • id_token_signing_alg_values_supported：RS256、RS384、RS512、ES256、ES384、ES512

    2.3.2 创建 OIDC IdP#

    控制台：IAM → 身份提供商 → 添加提供商 → OpenID Connect → 提供商 URL（区分大小写，仅能在当前账户唯一）+ 受众（客户端 ID）。

    CLI：

    aws iam create-open-id-connect-provider \
    --url https://token.actions.githubusercontent.com \
    --thumbprint-list <thumb> \
    --client-id-list sts.amazonaws.com

    API：CreateOpenIDConnectProvider。

    2.3.3 JWT 时钟偏差与 aud 规则#

    • AWS 在 OIDC 令牌 exp 时间外给 5 分钟时钟偏差容忍
    • 若 JWT 同时有 aud 和 azp，AWS STS 使用 azp 作为 aud
    • 创建 IdP 时填写的客户端 ID 必须与 JWT 中 aud 一致

    2.3.4 基于身份策略中的 ARN 匹配#

    OIDC 身份提供者 URL 带路径时，策略 Resource 必须包含完整路径（可用 /* 或 */? 通配）。不匹配会拒绝。

    2.4 GitHub Actions 到 AWS 示例#

    1. 创建 IAM OIDC IdP，URL 为 https://token.actions.githubusercontent.com，受众 sts.amazonaws.com
    2. 创建角色，信任策略：
    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": { "Federated": "arn:aws:iam::111122223333:oidc-provider/token.actions.githubusercontent.com" },
      "Action": "sts:AssumeRoleWithWebIdentity",
      "Condition": {
        "StringEquals": {
          "token.actions.githubusercontent.com:aud": "sts.amazonaws.com"
        },
        "StringLike": {
          "token.actions.githubusercontent.com:sub": "repo:my-org/my-repo:*"
        }
      }
    }
  ]
}
    1. 工作流中通过 aws-actions/configure-aws-credentials@v4 获取临时凭证

    3. SAML 2.0 联合#

    3.1 基本流程#

    典型 SSO 流程：

    1. 用户在企业门户（如 ADFS IdpInitiatedSignOn.aspx）登录
    2. 企业 IdP 验证用户身份
    3. IdP 生成 SAML 响应（包含断言）
    4. 客户端应用程序调用 AssumeRoleWithSAML（或 AWS 控制台登录端点）传递 SAML 断言
    5. AWS STS 返回临时凭证
    6. 用户访问 AWS 控制台或 API

    3.2 创建 IAM SAML 身份提供者#

    3.2.1 先决条件#

    从 IdP 获取元数据文件（XML），包含 issuer、公钥、终端节点。要求：

    • UTF-8 编码无 BOM
    • X.509 证书密钥长度至少 1024 位
    • 证书扩展项不得重复

    3.2.2 创建 IdP#

    控制台：IAM → 身份提供商 → 添加提供商 → SAML → 上传元数据文件。可选 SAML 加密（上传私钥，勾选“需要加密”）。

    CLI：

    aws iam create-saml-provider \
    --saml-metadata-document file://saml-metadata.xml \
    --name ExampleCorp

    API：CreateSAMLProvider。相关：UpdateSAMLProvider、DeleteSAMLProvider、ListSAMLProviders、TagSAMLProvider、UntagSAMLProvider。

    3.3 配置 SAML 断言#

    IdP 生成的 SAML 响应必须包含以下属性：

    Attribute Name 作用
    https://aws.amazon.com/SAML/Attributes/Role arn:aws:iam::<account>:role/<role-name>,arn:aws:iam::<account>:saml-provider/<provider-name> 对。可包含多个角色/提供商对，登录时用户选择
    https://aws.amazon.com/SAML/Attributes/RoleSessionName 会话名（2–64 个字符，允许字母数字和 . , + = @ -，不能含空格）
    https://aws.amazon.com/SAML/Attributes/SessionDuration（可选） 控制台会话持续时间
    https://aws.amazon.com/SAML/Attributes/PrincipalTag:{TagKey}（可选） 将 SAML 属性作为会话标签传递给 AWS
    https://aws.amazon.com/SAML/Attributes/TransitiveTagKeys（可选） 声明可传递的会话标签键

    3.4 SAML 条件键#

    IAM 策略中可使用的 SAML 上下文键：

    • saml:aud：受众 ARN（对应 SAML recipient 属性）
    • saml:iss：IdP issuer
    • saml:sub：主体 NameID
    • saml:sub_type：persistent / transient（transient 每次会话不同，不可用作稳定标识）
    • saml:cn、saml:mail、saml:eduPerson*、saml:eduOrg* 等（映射 IdP 属性）

    断言样例：仅允许 staff 群组访问：

    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": { "Federated": "arn:aws:iam::111122223333:saml-provider/ExampleCorp" },
      "Action": "sts:AssumeRoleWithSAML",
      "Condition": {
        "StringEquals": {
          "saml:aud": "https://signin.aws.amazon.com/saml",
          "saml:edupersonaffiliation": "staff"
        }
      }
    }
  ]
}

    3.5 AWS 控制台 SSO 流程#

    使用 https://signin.aws.amazon.com/saml 端点，IdP POST 一个 SAML 响应后 AWS 代替用户调用 API 并重定向浏览器到控制台。默认端点是全局，失效转移可参考区域端点策略。

    ADFS/Azure AD 门户 URL 格式：

    https://ADFSServiceName/adfs/ls/IdpInitiatedSignOn.aspx

    4. 获取联合令牌（GetFederationToken）#

    用于自定义身份凭证代理：

    1. 代理使用 IAM 用户凭证调用 sts:GetFederationToken，传递联合用户名与临时策略
    2. AWS STS 返回令牌
    3. 代理构造登录 URL 将用户重定向到控制台，或返回给应用使用

    注意：GetFederationToken 登录的会话不需要 MFA。

    5. SAML 联合故障排除#

    常见问题：

    错误 可能原因
    Invalid SAML response 元数据文件含 BOM、证书链顺序错乱、时钟偏差过大
    Signature invalid 元数据中公钥与实际响应签名不匹配
    RoleSessionName must match 属性值含非法字符或空格
    NotAuthorized 信任策略中 saml:aud 或其他条件未通过

    浏览器中抓取 SAML 响应：使用开发者工具 Network 标签中的 SAML POST 请求的 SAMLResponse 参数（Base64 编码）。PowerShell 解码：

    [System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String("<base64>"))

    6. 出站联合（AWS to External）#

    AWS 也可以作为 OIDC 颁发方，为特定工作负载颁发 JWT 令牌，用于访问外部服务。相关 API：

    • GetWebIdentityToken：为工作负载生成 JWT 令牌
    • 令牌声明包含 AWS 身份、会话上下文

    外部服务验证 JWT 签名后即可根据 AWS 身份授权。可在 IAM 策略中使用 sts: 条件键控制谁能获取 JWT。

    7. 客户自定义身份代理#

    当企业目录不兼容 SAML 且不想使用 IAM Identity Center 时，构建自定义身份代理：

    1. 用户登录企业身份系统
    2. 代理根据用户属性决定授权
    3. 代理调用 sts:GetFederationToken 或 sts:AssumeRole 获取临时凭证
    4. 构造控制台登录 URL（调用 https://signin.aws.amazon.com/federation 接口）
    5. 用户跳转进入 AWS 控制台

    登录 URL 生成需要 Action=getSigninToken + SessionDuration + Session（JSON 凭证），返回签名令牌后构造 Action=login URL。
