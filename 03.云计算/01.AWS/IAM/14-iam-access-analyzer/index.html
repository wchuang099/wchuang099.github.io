    1. Access Analyzer 功能总览#

    功能 说明 是否收费
    外部访问分析 识别组织/账户中与外部实体共享的资源 免费
    内部访问分析 识别组织/账户内哪些主体可以访问指定关键资源 按监控资源数按月
    未使用访问分析 识别未使用的 IAM 角色、用户、密码、访问密钥、权限 按监控的 IAM 用户/角色数按月
    策略验证 语法与 AWS 最佳实践检查，100+ 条规则 免费
    自定义策略检查 对照指定安全标准校验策略 按 API 请求次数
    策略生成 基于 CloudTrail 活动生成最小权限策略 免费

    底层使用 Zelkova 自动推理引擎，将 IAM 策略转换为逻辑等价语句后运行可满足性模理论求解器。

    2. 外部访问分析器#

    2.1 核心概念#

    • 信任区域：创建分析器时选择的组织或账户。区域内的主体均视为可信
    • 监控资源类型（2026 年）：
      • S3 存储桶、S3 目录存储桶
      • IAM 角色
      • KMS 密钥
      • Lambda 函数与层
      • SQS 队列
      • Secrets Manager 密钥
      • SNS 主题
      • EBS 卷/快照
      • RDS 数据库/集群快照
      • ECR 存储库
      • EFS 文件系统
      • DynamoDB 表/Streams
    • 区域性：必须在每个需要分析外部访问的区域创建分析器
    • 每次策略变更约 30 分钟内重新分析，定期扫描周期为 24 小时

    2.2 创建外部访问分析器#

    控制台：IAM → Access Analyzer → 分析器设置 → 创建分析器 → 资源分析 - 外部访问 → 选择当前账户或当前组织作为信任区域 → 提交。

    创建后自动生成服务相关角色 AWSServiceRoleForAccessAnalyzer。

    2.3 调查发现状态#

    状态 含义
    Active（活跃） 调查发现有效
    Archived（归档） 手动/规则归档，仍可查询
    Resolved（已解决） 生成调查发现的访问已被移除

    2.4 调查发现字段#

    • 资源（类型、ARN、账户）
    • 外部主体（账户、IAM 角色、联合身份、匿名等）
    • 授予的操作、条件
    • 首次发现时间与上次更新时间
    • 可选状态：新、活跃、已归档、已解决

    2.5 再次扫描#

    修改策略后可能需要约 30 分钟刷新，手动 Rescan：

    • 控制台：Finding 详情页 → Rescan
    • API：StartResourceScan

    3. 内部访问分析器#

    • 创建时需指定要监控的具体资源
    • 范围：组织或账户
    • 评估所有基于身份策略、基于资源策略、SCP、RCP、权限边界
    • 调查发现仅通过 ListFindingsV2 API 获取
    • 按监控资源数量每月计费
    • 限制：组织内主体（IAM 用户+角色）超过 70,000 时无法生成内部访问调查发现
    • 每个组织仅能创建一个组织级内部访问分析器

    监控资源类型（2026 年）：

    • S3 存储桶、目录存储桶
    • RDS 数据库/集群快照
    • DynamoDB 表/Streams

    初次扫描可能需要几分钟到数小时，之后每 24 小时扫描一次。

    4. 未使用访问分析器#

    4.1 支持的检测类型#

    • 未使用角色：指定使用窗口内无访问活动
    • 未使用用户访问密钥、密码：使用窗口内未使用
    • 未使用权限：角色身份策略中未用过的服务级或操作级权限

    4.2 不包含的对象#

    • 服务相关角色（AWS 自动管理，不分析）
    • 超过支持阈值的 IAM 操作（参考文档中 “上次访问 IAM 操作信息的服务和操作”）

    4.3 创建#

    aws accessanalyzer create-analyzer \
    --analyzer-name UnusedAccess \
    --type ACCOUNT_UNUSED_ACCESS \
    --configuration '{"unusedAccess":{"unusedAccessAge":90}}'

    4.4 周期#

    • 扫描范围由 unusedAccessAge 控制（天数）
    • 定期重新评估，如身份再次使用则标记为已解决

    5. 查看与处理调查发现#

    5.1 控制台#

    • 汇总控制面板：展示公共、跨账户、未使用、内部访问调查发现分布
    • 按资源类型、状态、分析器筛选
    • 选中调查发现可查看详情、Rescan 或归档

    5.2 归档规则（Archive Rule）#

    按条件自动归档已知安全的共享发现，避免告警噪声：

    aws accessanalyzer create-archive-rule \
    --analyzer-name ExternalAccess \
    --rule-name IgnoreKnownPartnerAccount \
    --filter '{"principalAccount":{"eq":["999999999999"]}}'

aws accessanalyzer list-archive-rules --analyzer-name ExternalAccess
aws accessanalyzer update-archive-rule ...
aws accessanalyzer delete-archive-rule ...

    5.3 API#

    操作 用途
    CreateAnalyzer / DeleteAnalyzer / UpdateAnalyzer 管理分析器
    ListFindings / ListFindingsV2 列出调查发现
    GetFinding 获取单条详情
    UpdateFindings 修改调查发现状态（归档、激活）
    StartResourceScan 手动重新扫描
    GetFindingsStatistics 获取统计信息

    6. 事件集成#

    6.1 EventBridge#

    IAM Access Analyzer 发现、归档、解决调查发现事件会发送到默认 EventBridge 事件总线，可触发 SNS、Lambda、Step Functions：

    {
  "source": ["aws.access-analyzer"],
  "detail-type": ["Access Analyzer Finding"]
}

    6.2 Security Hub CSPM#

    开启 Security Hub CSPM 后，IAM Access Analyzer 调查发现自动同步，便于统一合规视图。

    6.3 CloudTrail#

    所有 Access Analyzer API 调用记录在 CloudTrail 中，包括 Create/Update/DeleteAnalyzer、StartPolicyGeneration、ValidatePolicy 等。

    7. 策略验证与自定义策略检查#

    7.1 策略验证（免费）#

    • 创建或编辑策略时自动运行
    • 返回四类：错误（Errors）、安全警告（Security warnings）、一般警告（Warnings）、建议（Suggestions）

    CLI：

    aws accessanalyzer validate-policy \
    --policy-type IDENTITY_POLICY \
    --policy-document file://p.json \
    --validate-policy-resource-type AWS::S3::Bucket

    7.2 自定义策略检查（收费）#

    按 API 请求次数计费。在 CI/CD 中使用，避免部署前引入过度授权。

    常见检查：

    • CheckNoNewAccess：新旧策略相比不引入新权限
    • CheckAccessNotGranted：策略不允许指定的敏感操作
    • CheckNoPublicAccess：策略不授予公网访问
    aws accessanalyzer check-no-new-access \
    --new-policy-document file://new.json \
    --existing-policy-document file://old.json \
    --policy-type IDENTITY_POLICY

aws accessanalyzer check-access-not-granted \
    --policy-document file://p.json \
    --access '[{"actions":["s3:DeleteBucket"]}]' \
    --policy-type IDENTITY_POLICY

    7.3 策略生成#

    基于 CloudTrail 生成最小权限策略：

    # 启动策略生成
aws accessanalyzer start-policy-generation \
    --policy-generation-details '{"principalArn":"arn:aws:iam::111:role/MyRole"}' \
    --cloud-trail-details file://trail.json

# 查看策略生成列表
aws accessanalyzer list-policy-generations

# 获取结果
aws accessanalyzer get-generated-policy --job-id <id>

# 取消
aws accessanalyzer cancel-policy-generation --job-id <id>

    支持的服务识别操作级细节（大多数服务）；其他服务给出模板，需手动补充 Action 与 Resource。

    8. 委托管理员#

    组织场景下，可将某个成员账户指定为 IAM Access Analyzer 委托管理员：

    aws organizations register-delegated-administrator \
    --service-principal access-analyzer.amazonaws.com \
    --account-id 222222222222

    委托管理员可创建组织级分析器并集中管理调查发现。

    9. 权限要求#

    9.1 AWS 托管策略#

    • IAMAccessAnalyzerFullAccess：管理员访问（含创建 SLR 权限）
    • IAMAccessAnalyzerReadOnlyAccess：只读，需额外策略才能查看调查发现

    9.2 控制台所需最小权限#

    查看控制面板：GetAnalyzer、ListAnalyzers、GetFindingsStatistics。

    9.3 服务相关角色#

    AWSServiceRoleForAccessAnalyzer（附加托管策略 AccessAnalyzerServiceRolePolicy）提供只读访问以分析资源。

    10. 预览访问#

    10.1 作用#

    在修改资源策略前，预览会造成哪些外部访问调查发现，避免误开放。

    10.2 API#

    aws accessanalyzer create-access-preview \
    --analyzer-arn <arn> \
    --configurations file://preview-config.json

aws accessanalyzer get-access-preview \
    --analyzer-arn <arn> --access-preview-id <id>

aws accessanalyzer list-access-preview-findings \
    --analyzer-arn <arn> --access-preview-id <id>

    控制台：S3 控制台编辑桶策略时自动提示“访问此存储桶的人”。

    11. 常见使用建议#

    • 在每个使用 AWS 资源的区域创建外部访问分析器
    • 组织根账户或委托管理员创建组织级分析器，覆盖所有成员账户
    • 使用归档规则屏蔽已知业务合作伙伴账户，减少噪声
    • 定期审阅未使用访问调查发现，清理多余权限
    • 在 CI/CD 管道中加入 check-no-new-access 防止权限扩张
    • 启用 Security Hub CSPM 集成，统一处理
