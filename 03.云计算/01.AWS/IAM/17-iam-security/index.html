    1. 责任共担模型#

    维度 责任方
    云的安全性 AWS 负责基础设施、物理数据中心、全球网络
    云中的安全性 用户负责：数据敏感性、合规需求、法律法规、IAM 配置

    AWS 为 IAM 提供合规性计划认证（SOC、PCI、ISO、FedRAMP 等），详细列表见 AWS Compliance Programs。

    2. AWS 安全凭证类别#

    2.1 用户类型#

    • AWS 账户根用户：创建账户时形成，完全访问权限
    • IAM Identity Center 用户：在 IAM Identity Center 管理的人力身份
    • 联合主体：外部 IdP 通过联合身份验证接入
    • IAM 用户：直接在 IAM 中创建

    2.2 凭证类型#

    • 长期：根用户密码、IAM 用户密码与访问密钥
    • 临时：IAM 角色、IAM Identity Center、STS 联合主体

    临时凭证的优点：

    • 短期、动态生成
    • 不必嵌入应用
    • 是角色与联合身份基础
    • 过期自动失效

    2.3 访问密钥 ID 前缀识别#

    • AKIA*：长期 IAM 用户或根用户密钥
    • ASIA*：STS 临时访问密钥
    • ABIA*：STS 服务持有者令牌

    3. 安全注意事项#

    • 不使用根用户执行日常任务
    • 强制 MFA（根用户、IAM 用户、IAM Identity Center 用户）
    • 不能用 IAM 策略显式拒绝根用户，只能用组织 SCP
    • 凭证作用域限定于当前账户，不跨账户共享
    • 修改或撤销策略是响应安全事件的首选方式
    • 紧急访问 IAM 用户凭证要保存在安全位置

    4. 编程访问选择#

    用户类型 推荐方案
    IAM Identity Center 用户 通过 aws configure sso 或 SSO 凭证获取临时凭证
    IAM 用户 使用 aws login 的控制台短期凭证；或担任 IAM 角色
    人力身份（联合） AWS CLI v2 + IAM Identity Center
    应用程序运行在 AWS IAM 角色（EC2 实例配置文件、ECS 任务角色、EKS IRSA、Lambda 执行角色）
    应用程序运行在 AWS 外 IAM Roles Anywhere（X.509）、SAML/OIDC 联合、ECS Anywhere

    4.1 长期访问密钥替代方案#

    • 代码或仓库里禁止嵌入密钥：使用 AWS Secrets Manager
    • 使用 IAM 角色生成临时凭证
    • 使用控制台凭证 + aws login
    • 使用 AWS CloudShell（浏览器预先验证）
    • 使用 IAM Identity Center 权限集替代长期 IAM 用户
    • 计算资源分配 IAM 角色（实例配置文件、任务角色）

    5. 安全审核指南#

    5.1 何时审核#

    • 定期（季度、半年）
    • 组织变动（员工离职、角色调整）
    • 停用或新增 AWS 服务
    • 部署或下线应用程序
    • 怀疑账户被未授权访问

    5.2 审核原则#

    • 全面覆盖所有身份、角色、策略、凭证
    • 对不熟悉的配置调研其业务目的
    • 保持配置简单：组管理权限、一致命名、简洁策略

    5.3 账户凭证审核步骤#

    1. 根用户访问密钥：如有应删除
    2. 根用户 MFA：必须启用
    3. 根用户密码与邮箱：保持可访问、强密码
    4. 长期凭证：定期轮换

    5.4 IAM 用户审核#

    1. 列出所有用户，删除不需要的
    2. 从无需加入的组移除用户
    3. 审核组策略（见策略审核）
    4. 删除不用的密码、访问密钥、SSH 密钥、服务特定凭证
    5. 使用凭证报告查看最近使用时间
    6. 对确需长期凭证的场景轮换密钥

    生成凭证报告：

    aws iam generate-credential-report
aws iam get-credential-report --output text --query Content | base64 -d > report.csv

    5.5 IAM 组审核#

    • 删除未使用的组
    • 检查组内成员与附加策略
    • 使用上次访问信息识别无效权限

    5.6 IAM 角色审核#

    • 删除不再使用的角色
    • 审核信任策略：信任对象是否仍合理
    • 审核权限策略
    • 清理长时间未被担任的角色
    • 检查 ExternalId 配置（第三方角色）

    5.7 身份提供者审核#

    • SAML：元数据是否仍有效，证书是否即将过期
    • OIDC：JWKS 端点是否可访问
    • 清理不再使用的 IdP

    5.8 移动应用审核#

    • 确认使用 Cognito 身份池代替长期密钥
    • 验证角色权限限定在最小集
    • 确认未将长期密钥硬编码到客户端

    5.9 策略审核要点#

    • 使用 IAM Access Analyzer 验证语法和最佳实践
    • 使用最低权限原则
    • 注意通配 * 的滥用
    • 识别 Admin、Power User 级别的过度授权
    • 检查条件键和标签约束

    6. 数据保护#

    6.1 静态与传输中的数据保护#

    • IAM 管理的凭证在 AWS 内部加密存储
    • AWS 服务间使用 TLS 1.2+ 加密传输
    • 建议始终通过 HTTPS 调用 API：条件 aws:SecureTransport

    6.2 客户关注事项#

    • 不在代码仓库提交访问密钥
    • 使用 AWS Secrets Manager 存储数据库凭证、API Token
    • 使用 KMS 加密静态资源（S3、RDS、EBS）
    • 考虑区域数据驻留需求

    7. 日志记录与监控#

    7.1 CloudTrail#

    • 记录所有 IAM 和 STS API 调用
    • 事件包括调用者、时间、操作、资源、源 IP
    • 支持管理事件、数据事件、Insight 事件
    • 用于识别异常：根登录、异地访问、权限变化

    示例事件类型：ConsoleLogin、AssumeRole、AssumeRoleWithSAML、CreatePolicy、AttachRolePolicy、AssumeRoot 等。

    7.2 CloudWatch / EventBridge#

    • 基于 CloudTrail 事件构建实时告警
    • 根用户登录、密钥创建、权限变更等敏感事件应触发通知

    7.3 IAM Access Analyzer#

    • 持续监控外部共享与未使用访问
    • 发现安全风险自动生成调查发现

    7.4 AWS Config#

    • 托管规则监控 IAM 配置合规
    • 示例规则：iam-root-access-key-check、mfa-enabled-for-iam-console-access、iam-user-no-policies-check

    7.5 Security Hub CSPM#

    • 统一合规视图
    • 集成 IAM Access Analyzer、Inspector、GuardDuty 等
    • 支持 AWS Foundational Security Best Practices、CIS AWS Foundations、PCI DSS 等框架

    8. 合规性验证#

    IAM 覆盖的合规性计划示例：

    • SOC 1、2、3
    • PCI DSS Level 1
    • ISO 27001、27017、27018、9001
    • FedRAMP、DoD SRG
    • HIPAA BAA
    • GDPR（共同责任）

    详细列表访问 AWS Compliance Programs 页面。

    9. 故障恢复能力#

    • IAM 数据跨多个 AWS 全球数据中心复制
    • IAM STS 提供区域端点与全局端点，建议使用区域端点
    • 凭证全球通用，临时凭证可在任何区域使用
    • 构建弹性时为 STS 设置回退端点，避免区域性故障阻断所有业务

    10. 基础设施安全#

    • IAM 与 STS 通过 HTTPS API 暴露
    • 支持 VPC 接口端点（AWS PrivateLink）减少公网依赖
    • 支持双堆栈端点（IPv6）
    • 接口端点策略可精细控制允许的 IAM API

    11. 配置与漏洞分析#

    • AWS 负责底层安全与漏洞修复
    • 客户负责配置管理：最小权限、凭证轮换、MFA、标签治理
    • 使用 IAM Access Analyzer、AWS Config、Security Hub CSPM 定期扫描

    12. 常用 AWS 托管策略#

    12.1 IAM / Access Analyzer 管理#

    • IAMFullAccess
    • IAMReadOnlyAccess
    • IAMUserChangePassword
    • IAMSelfManageServiceSpecificCredentials
    • IAMAccessAnalyzerFullAccess
    • IAMAccessAnalyzerReadOnlyAccess
    • AccessAnalyzerServiceRolePolicy
    • IAMAuditRootUserCredentials
    • IAMCreateRootUserPassword
    • IAMDeleteRootUserCredentials

    12.2 特权任务策略（AssumeRoot 专用）#

    • S3UnlockBucketPolicy
    • SQSUnlockQueuePolicy

    13. IAM 外部的安全功能#

    • AWS KMS：密钥管理与加密
    • AWS Secrets Manager：敏感凭证存储与自动轮换
    • AWS Certificate Manager：SSL/TLS 证书托管
    • AWS WAF / Shield：应用层防护
    • Amazon GuardDuty：威胁检测
    • AWS Security Hub CSPM：安全标准评估
    • AWS Macie：敏感数据发现
    • AWS Firewall Manager：统一防火墙策略
    • AWS IoT Core / Device Defender：物联网安全

    14. 安全事件响应建议#

    14.1 凭证泄露#

    1. 立即停用泄露的访问密钥：aws iam update-access-key --status Inactive
    2. 创建新密钥并迁移应用
    3. 删除旧密钥
    4. 审查 CloudTrail 判定滥用范围
    5. 修改相关受影响资源的凭证或秘密
    6. 旋转密码、重新颁发证书

    14.2 根凭证泄露#

    1. 立刻重置根密码并轮换 MFA
    2. 删除所有根访问密钥
    3. 审查账户活动
    4. 联系 AWS Support
    5. 若为 Organizations 成员，管理账户可通过 AssumeRoot 执行清理

    14.3 角色被滥用#

    1. 立即附加拒绝策略或修改信任策略
    2. 使用 aws:TokenIssueTime 拒绝早期发行的会话凭证
    3. 审计 CloudTrail 全部相关事件
    4. 审查谁有 iam:PassRole、sts:AssumeRole 权限
