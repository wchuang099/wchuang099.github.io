    1. IAM 用户概览#

    IAM 用户是 AWS 账户内代表人员或工作负载的实体，拥有名称和凭证。IAM 用户不同于根用户：根用户是账户创建时的初始身份，拥有完全访问权限。

    1.1 IAM 用户标识符#

    • 易记名称：创建时指定，显示在控制台和 ARN 中（避免使用个人身份信息）
    • ARN：arn:aws:iam::account-ID-without-hyphens:user/Richard，全 AWS 唯一
    • 唯一标识符：仅在通过 API/CLI 创建时返回，控制台不显示

    1.2 凭证类型#

    凭证 用途
    控制台密码 登录 AWS 管理控制台；也可搭配 aws login 对 CLI/SDK 进行身份验证
    访问密钥 编程方式访问 AWS
    SSH 公钥 向 CodeCommit 身份验证
    服务器证书 SSL/TLS 连接（建议用 ACM 替代）
    Git/服务特定凭证 CodeCommit、Amazon Keyspaces 等服务

    控制台创建用户必须至少指定控制台密码或访问密钥；CLI/API 创建的用户默认无凭证。

    1.3 权限原则#

    • 新用户默认无任何权限
    • 通过直接附加策略或加入用户组授权
    • 可通过权限边界限制用户最大权限（不授予，仅限制）

    2. 登录 AWS#

    2.1 登录 URL#

    https://12-digit-AWS-account-ID-or-alias.signin.aws.amazon.com/console

    通用入口 https://console.aws.amazon.com/ 需手动输入账户 ID 或别名。

    建议手动粘贴登录 URL 到书签栏，不要用浏览器书签功能（重定向会覆盖 URL）。同一浏览器最多可同时登录 5 个身份。

    2.2 MFA 登录流程#

    • 多个 MFA 设备：用户选择使用哪一个完成验证
    • FIDO 安全密钥：直接点击密钥完成，丢失或损坏需管理员停用
    • 虚拟 MFA 设备：输入 APP 生成的 6 位代码，多次失败后可引导重新同步
    • 硬件 TOTP 令牌：输入令牌显示的代码，支持重新同步

    AWS STS GetFederationToken 调用产生的控制台访问不需要 MFA。

    2.3 CloudTrail 日志区域#

    • 登录到区域感知控制台主页 → 区域 CloudTrail 日志
    • 登录到 S3 等非区域控制台 → 全局登录端点 → 全局 CloudTrail 日志
    • 使用 ?region=xxx 参数可强制登录到特定区域端点

    3. 创建 IAM 用户#

    流程：

    1. 控制台、CLI、PowerShell、API 创建用户
    2. 根据访问类型创建凭证（仅创建必要的）
    3. 授予权限（优先通过组管理）；可加权限边界
    4. 可选标签
    5. 提供登录信息（URL + 密码 + 账户 ID/别名）
    6. 可选配置 MFA
    7. 可选授予自我管理凭证权限

    4. 查看 IAM 用户#

    控制台：IAM → 用户列表、用户组 → 用户页、用户详情 → 组选项卡。

    CLI：

    # 列出所有用户
aws iam list-users

# 列出组内用户
aws iam get-group --group-name <group>

# 列出用户所属组
aws iam list-groups-for-user --user-name <user>

    API：ListUsers、GetGroup、ListGroupsForUser。

    5. 重命名 IAM 用户#

    • 仅支持 CLI/API，控制台不支持
    • 策略、组成员资格、唯一 ID 保持不变
    • 将用户作为主体的资源或角色策略会自动更新
    • 将用户作为资源的策略需手动更新 ARN
    aws iam update-user --user-name OldName --new-user-name NewName

    API：UpdateUser。

    6. 删除 IAM 用户#

    6.1 控制台删除#

    控制台会自动删除：用户标识符、组成员资格、密码、内联策略、关联 MFA 设备。托管策略仅分离不删除。有激活的访问密钥需先停用。

    6.2 CLI 删除流程#

    # 1. 删除登录密码
aws iam delete-login-profile --user-name <user>

# 2. 删除访问密钥
aws iam list-access-keys --user-name <user>
aws iam delete-access-key --user-name <user> --access-key-id <id>

# 3. 删除签名证书
aws iam list-signing-certificates --user-name <user>
aws iam delete-signing-certificate --user-name <user> --certificate-id <id>

# 4. 删除 SSH 公钥
aws iam list-ssh-public-keys --user-name <user>
aws iam delete-ssh-public-key --user-name <user> --ssh-public-key-id <id>

# 5. 删除服务特定凭证
aws iam list-service-specific-credentials --user-name <user>
aws iam delete-service-specific-credential --user-name <user> \
    --service-specific-credential-id <id>

# 6. 停用 MFA
aws iam list-mfa-devices --user-name <user>
aws iam deactivate-mfa-device --user-name <user> --serial-number <arn>
aws iam delete-virtual-mfa-device --serial-number <arn>

# 7. 删除内联策略
aws iam list-user-policies --user-name <user>
aws iam delete-user-policy --user-name <user> --policy-name <name>

# 8. 分离托管策略
aws iam list-attached-user-policies --user-name <user>
aws iam detach-user-policy --user-name <user> --policy-arn <arn>

# 9. 从组中移除
aws iam list-groups-for-user --user-name <user>
aws iam remove-user-from-group --user-name <user> --group-name <group>

# 10. 删除用户
aws iam delete-user --user-name <user>

    6.3 停用替代删除#

    临时停用用户：附加一个显式拒绝所有的策略，保留凭证待日后恢复。

    永久拒绝：

    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Action": "*",
      "Resource": "*"
    }
  ]
}

    带时间窗口：

    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Action": "*",
      "Resource": "*",
      "Condition": {
        "DateGreaterThan": {"aws:CurrentTime": "2024-12-24T23:59:59Z"},
        "DateLessThan": {"aws:CurrentTime": "2025-02-28T23:59:59Z"}
      }
    }
  ]
}

    7. 控制台访问#

    • 禁用控制台访问：删除用户密码即可，保留有效的访问密钥仍能通过 CLI/API 访问
    • 通过 IAM 策略控制控制台操作权限，AWS 管理控制台 只能展示用户有权查看的资源
    • 账单信息、账户个人资料、安全凭证的访问控制取决于专项设置

    8. 密码管理#

    8.1 账户密码策略#

    • 不适用于根用户密码或访问密钥
    • 大多数设置（长度、复杂度）从下次修改密码时生效
    • 密码有效期立即生效，可能强制已有用户登录时修改
    • 无法设置登录失败锁定策略，建议搭配 MFA

    8.2 权限#

    操作 含义
    iam:GetAccountPasswordPolicy 查看账户密码策略
    iam:UpdateAccountPasswordPolicy 创建或修改自定义密码策略
    iam:DeleteAccountPasswordPolicy 恢复默认策略

    8.3 默认密码策略#

    • 长度 8–128 字符
    • 至少包含 3 种字符类型组合：大写、小写、数字、符号
    • 不得与账户名称或邮箱相同
    • 永不过期

    8.4 自定义密码策略选项#

    • 最小长度：6–128
    • 强度要求：大写、小写、数字、非字母数字字符
    • 过期时间：1–1095 天
    • 过期需管理员重置（勾选后用户无法自助修改已过期密码）
    • 允许用户修改自己密码（无需额外策略）
    • 防止密码重用：历史记忆 1–24 个

    CLI：

    aws iam update-account-password-policy --minimum-password-length 12 \
    --require-uppercase-characters --require-lowercase-characters \
    --require-numbers --require-symbols \
    --password-reuse-prevention 5 \
    --max-password-age 90 \
    --hard-expiry \
    --allow-users-to-change-password

aws iam get-account-password-policy
aws iam delete-account-password-policy

    8.5 管理用户密码#

    控制台：用户详情 → 安全凭证 → 控制台登录 → 启用/管理。可选项：

    • 自动生成或自定义密码（需符合账户策略）
    • 要求下次登录修改密码
    • 撤销活动控制台会话：附加一条基于 aws:TokenIssueTime 的拒绝策略
    aws iam get-login-profile --user-name <user>
aws iam create-login-profile --user-name <user> --password <p> --password-reset-required
aws iam update-login-profile --user-name <user> --password <p> --password-reset-required
aws iam delete-login-profile --user-name <user>

    撤销早于某时间的活动会话：

    aws iam put-user-policy --user-name <user> \
    --policy-name RevokeOldSessions \
    --policy-document '{
      "Version": "2012-10-17",
      "Statement": {
        "Effect": "Deny",
        "Action": "*",
        "Resource": "*",
        "Condition": {
          "DateLessThan": { "aws:TokenIssueTime": "2014-05-07T23:47:00Z" }
        }
      }
    }'

    8.6 允许用户修改自己的密码#

    两种方式：

    账户级统一开启（控制台 → 账户设置 → 密码策略 → 勾选 Allow users to change their own password）。所有用户自动获得 iam:ChangePassword 与 iam:GetAccountPasswordPolicy 权限。

    仅部分用户（附加策略）：

    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "iam:GetAccountPasswordPolicy",
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": "iam:ChangePassword",
      "Resource": "arn:aws:iam::*:user/${aws:username}"
    }
  ]
}

    用户控制台修改密码：右上角用户名 → 安全凭证 → 修改密码。

    CLI：aws iam change-password；API：ChangePassword。

    9. 访问密钥管理#

    9.1 基础概念#

    • 由访问密钥 ID（如 AKIAIOSFODNN7EXAMPLE）和秘密访问密钥（如 wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY）组成
    • 每个用户最多 2 个访问密钥（便于轮换）
    • 仅创建时可下载秘密密钥，无法恢复，务必保存

    9.2 管理权限#

    关键权限：

    • iam:CreateAccessKey、iam:ListAccessKeys、iam:UpdateAccessKey、iam:DeleteAccessKey
    • iam:GetAccessKeyLastUsed
    • 可选：iam:TagUser（用于编辑密钥描述）

    9.3 常用 CLI 操作#

    # 创建
aws iam create-access-key --user-name <user>

# 列出
aws iam list-access-keys --user-name <user>

# 停用/激活
aws iam update-access-key --user-name <user> --access-key-id <id> --status Inactive
aws iam update-access-key --user-name <user> --access-key-id <id> --status Active

# 删除
aws iam delete-access-key --user-name <user> --access-key-id <id>

# 上次使用时间
aws iam get-access-key-last-used --access-key-id <id>

    API 对应：CreateAccessKey、ListAccessKeys、UpdateAccessKey、DeleteAccessKey、GetAccessKeyLastUsed。

    9.4 查找访问密钥所属用户#

    IAM 控制台顶部搜索访问密钥 ID → 进入用户详情。

    9.5 密钥轮换#

    1. 为用户创建第二个密钥
    2. 更新应用使用新密钥
    3. 在 GetAccessKeyLastUsed 中确认旧密钥不再使用
    4. 停用旧密钥观察
    5. 确认无影响后删除旧密钥

    9.6 保护访问密钥#

    • 不嵌入代码或公开仓库
    • 不分享，不通过公共邮件发送
    • 通过 AWS SDK 从 ~/.aws/credentials、环境变量、IAM 角色（EC2/ECS/Lambda）中读取
    • 使用强 MFA 保护账户
    • 定期轮换
    • 使用 IAM Identity Center 或 IAM Roles Anywhere 代替

    10. IAM 服务账户#

    若必须用 IAM 用户作为“服务账户”给应用使用：

    • 不要嵌入密钥，改用 AWS SDK 配置的位置加载
    • 考虑替代方案：IAM 角色 + 临时凭证
    • 轮换机制要就位

    11. 其他凭证#

    11.1 CodeCommit 凭证#

    • HTTPS Git 凭证（用户名+密码）
    • SSH 公钥
    • 服务特定凭证
    • 访问密钥配合 git-remote-codecommit 使用临时凭证

    11.2 Amazon Keyspaces 凭证#

    • 服务特定凭证（长期）
    • 推荐：临时凭证 + SigV4 身份验证插件

    11.3 服务器证书#

    • 优先使用 AWS Certificate Manager
    • 仅在 ACM 不支持的区域使用 IAM 上传 SSL/TLS 证书
    • 凭证报告可查询证书过期情况

    12. 凭证报告#

    凭证报告列出账户所有 IAM 用户及其凭证状态：密码、访问密钥、MFA、SSH 公钥等。

    aws iam generate-credential-report
aws iam get-credential-report

    报告以 CSV 返回，需 Base64 解码。可用于审计、合规性、密钥轮换跟踪。
