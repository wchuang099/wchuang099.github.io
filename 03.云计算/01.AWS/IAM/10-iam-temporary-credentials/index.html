    1. 临时安全凭证概述#

    AWS Security Token Service（AWS STS）颁发临时安全凭证，有以下特点：

    • 生命周期短（几分钟到几小时），过期后无法使用
    • 动态生成，不与 IAM 用户永久存储
    • 凭证组成：AccessKeyId、SecretAccessKey、SessionToken

    相比长期凭证的优势：

    • 无需将密钥嵌入应用
    • 无需为每个用户创建 IAM 身份
    • 到期自动失效，免除轮换与撤销

    1.1 典型场景#

    • 联合身份（SAML、OIDC、企业 IdP）
    • 跨账户角色委派
    • EC2 实例角色
    • 应用程序调用 AssumeRole
    • MFA 保护的敏感 API 操作

    1.2 AWS STS 与区域#

    • 全球端点：https://sts.amazonaws.com
    • 区域端点：https://sts.<region>.amazonaws.com
    • 凭证全球有效，选择就近区域可降低延迟
    • 默认使用全局端点，生产建议启用区域端点避免跨区故障
    • sts:AssumeRoot 不支持全局端点，必须调用区域端点

    2. AWS STS API 对比#

    API 调用者 生命周期（最小 / 最大 / 默认） MFA 支持 会话策略 限制
    AssumeRole IAM 用户或已有临时凭证的角色 15 分钟 / 最大会话时长 / 1 小时 是 是 不能调用 GetFederationToken、GetSessionToken
    AssumeRoleWithSAML 任意用户（传递 SAML 响应） 同上 否 是 同上
    AssumeRoleWithWebIdentity 任意用户（传递 OIDC JWT） 同上 否 是 同上
    GetFederationToken IAM 用户 / 根用户 15 分钟 / 36 小时 / 12 小时（根 1 小时） 否 是 不能调用 IAM API（控制台除外）；只能调 GetCallerIdentity；可用于 SSO 登录控制台
    GetSessionToken IAM 用户 / 根用户 同 GetFederationToken 是 否 除非有 MFA 否则不能调 IAM API；仅能调 AssumeRole 或 GetCallerIdentity；不能 SSO 控制台

    2.1 最大会话时长#

    • 默认 1 小时
    • 角色可设置 1–12 小时（DurationSeconds 900–43200）
    • 角色链强制上限 1 小时（即便单角色设置更长）
    • GetCallerIdentity 无需权限即可调用

    2.2 会话策略#

    • 仅对当前会话生效，进一步缩减权限
    • 最终权限 = 角色身份策略 ∩ 会话策略
    • PackedPolicySize 响应字段指示策略大小接近上限的比例

    3. 请求临时凭证的 API#

    3.1 AssumeRole#

    调用方必须有有效 AWS 凭证。典型用途：跨账户访问、MFA 保护、自定义联合代理。

    CLI：

    aws sts assume-role \
    --role-arn arn:aws:iam::123456789012:role/demo \
    --role-session-name John-session \
    --duration-seconds 1800 \
    --external-id 123ABC \
    --source-identity DevUser123 \
    --policy '{"Version":"2012-10-17","Statement":[{"Sid":"Stmt1","Effect":"Allow","Action":"s3:*","Resource":"*"}]}' \
    --tags Key=Project,Value=Pegasus Key=Cost-Center,Value=12345

    响应：

    <AssumeRoleResponse>
  <AssumeRoleResult>
    <Credentials>
      <AccessKeyId>AKIAIOSFODNN7EXAMPLE</AccessKeyId>
      <SecretAccessKey>wJalrXUtnFEMI/K7MDENG/bPxRfiCYzEXAMPLEKEY</SecretAccessKey>
      <SessionToken>AQoDYXdzEPT...</SessionToken>
      <Expiration>2026-07-15T23:28:33Z</Expiration>
    </Credentials>
    <AssumedRoleUser>
      <Arn>arn:aws:sts::123456789012:assumed-role/demo/John-session</Arn>
      <AssumedRoleId>ARO123EXAMPLE123:John-session</AssumedRoleId>
    </AssumedRoleUser>
  </AssumeRoleResult>
</AssumeRoleResponse>

    3.2 AssumeRoleWithSAML#

    • 传递 IdP 生成的 SAML 响应
    • 不需要 AWS 凭证
    • 不支持 MFA，SAML 断言自身携带验证信息
    aws sts assume-role-with-saml \
    --role-arn arn:aws:iam::111:role/SAMLRole \
    --principal-arn arn:aws:iam::111:saml-provider/ExampleCorp \
    --saml-assertion file://saml-response.b64 \
    --duration-seconds 3600

    3.3 AssumeRoleWithWebIdentity#

    • 传递 OIDC IdP 的 JWT（Login with Amazon、Facebook、Google、自建 OIDC）
    • 不需要签名和 AWS 凭证
    • JWT 过期前 5 分钟时钟偏差容忍
    aws sts assume-role-with-web-identity \
    --role-arn arn:aws:iam::111:role/WebIdRole \
    --role-session-name mobile-user \
    --web-identity-token <jwt>

    3.4 GetFederationToken#

    用于构建自定义身份代理。代理用 IAM 用户凭证调用，返回的令牌可用于登录控制台或编程访问：

    aws sts get-federation-token \
    --name FedUser \
    --duration-seconds 3600 \
    --policy file://session-policy.json

    根用户调用最大 1 小时；IAM 用户调用最大 36 小时。

    3.5 GetSessionToken#

    为已有 IAM 凭证生成临时会话（常用于 MFA 场景）：

    aws sts get-session-token \
    --duration-seconds 3600 \
    --serial-number arn:aws:iam::111:mfa/User \
    --token-code 123456

    3.6 GetCallerIdentity#

    返回当前调用方信息，无需权限即可执行。适合验证凭证是否生效：

    aws sts get-caller-identity

    输出示例：

    {
  "UserId": "AROAEXAMPLE:session",
  "Account": "123456789012",
  "Arn": "arn:aws:sts::123456789012:assumed-role/demo/session"
}

    4. 担任角色的方法#

    • 控制台切换角色（IAM 用户、IAM Identity Center 用户）
    • CLI/SDK AssumeRole
    • AWS 配置文件：~/.aws/config 中设置 source_profile 与 role_arn
    • SAML/OIDC 联合
    • 服务主体代入（自动，如 EC2 实例角色、Lambda 执行角色）

    5. 服务持有者令牌（Service Bearer Token）#

    部分服务（如 AWS CodeArtifact）使用持有者令牌而非 SigV4：

    • 服务代为调用 STS 获取令牌
    • AccessKeyId 以 ABIA 前缀开头，便于在 CloudTrail 中识别
    • 持有者令牌仅对生成它的服务和区域有效
    • 需要的权限：sts:GetServiceBearerToken
    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowServiceBearerToken",
      "Effect": "Allow",
      "Action": "sts:GetServiceBearerToken",
      "Resource": "*"
    }
  ]
}

    CodeArtifact 示例：

    aws codeartifact get-authorization-token \
    --domain my-domain --domain-owner 111122223333 \
    --query authorizationToken --output text

# 或者自动登录 NPM / pip / Maven
aws codeartifact login --tool npm --domain my-domain --repository my-repo

    6. 使用 MFA 保护 API 访问#

    在角色信任策略或资源策略中要求 MFA：

    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": { "AWS": "arn:aws:iam::111:user/Admin" },
      "Action": "sts:AssumeRole",
      "Condition": {
        "Bool": { "aws:MultiFactorAuthPresent": "true" },
        "NumericLessThan": { "aws:MultiFactorAuthAge": "3600" }
      }
    }
  ]
}
    • aws:MultiFactorAuthPresent：是否使用 MFA 完成认证
    • aws:MultiFactorAuthAge：距离完成 MFA 的秒数

    7. 会话标签#

    7.1 传递会话标签#

    调用 AssumeRole* 时可传递 Tags 参数（单值键值对，AWS STS 不支持多值）：

    aws sts assume-role --tags Key=Project,Value=Peg Key=CostCenter,Value=987654 \
    --transitive-tag-keys Project \
    ...

    7.2 策略使用#

    通过 ${aws:PrincipalTag/<key>} 在策略中动态匹配，用于 ABAC：

    {
  "Condition": {
    "StringEquals": {
      "aws:ResourceTag/Project": "${aws:PrincipalTag/Project}"
    }
  }
}

    7.3 可传递标签#

    设置为可传递后，标签随角色链继续携带。其他标签仅当前会话有效。

    8. 源身份（SourceIdentity）#

    sts:AssumeRole 等操作可通过 SourceIdentity 参数传递不可变的调用方身份信息，写入 CloudTrail 用于追踪。策略条件键：

    • sts:SourceIdentity：要求请求中必须包含源身份
    • aws:SourceIdentity：在后续调用中检查源身份
    {
  "Effect": "Deny",
  "Action": "sts:AssumeRole",
  "Resource": "*",
  "Condition": {
    "StringEquals": {
      "sts:SourceIdentity": ""
    }
  }
}

    9. 停用对临时凭证的所有访问#

    当临时凭证泄露需要立即撤销时，无法单独吊销凭证本身。替代策略：向原角色附加拒绝策略，限制 aws:TokenIssueTime 早于当前时间的所有操作：

    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Action": "*",
      "Resource": "*",
      "Condition": {
        "DateLessThan": {
          "aws:TokenIssueTime": "2026-05-07T00:00:00Z"
        }
      }
    }
  ]
}

    控制台提供“撤销会话”按钮自动执行此操作。

    10. 最佳实践#

    • 优先临时凭证替代长期密钥
    • 使用 IAM Identity Center 或 IAM Roles Anywhere
    • 将 STS 调用发往区域端点，提升可用性和延迟
    • 为角色设置最小的最大会话时长
    • 结合 ExternalId、MFA、源身份增强安全
    • 定期审查 CloudTrail 中的 STS 事件
