    1. AWS CLI介绍#

    1.1 为什么要学命令行#

    使用AWS Console（网页控制台）完成操作效率低下。以在10个Region各创建一个S3存储桶为例：

    • Console操作：每个Region需要7步鼠标操作，10个Region共需70次点击
    • CLI操作：3行脚本，5秒完成
    for region in us-east-1 us-west-2 eu-west-1 ...; do
  aws s3 mb s3://my-bucket-$region --region $region
done

    CLI的核心价值：效率与可重复性。在真实工作中，80%的AWS操作通过CLI或SDK完成。

    1.2 AWS三种交互方式对比#

    方式 使用方法 优点 缺点 适合场景
    Console 浏览器操作 直观，适合探索 效率低，难重复 初次了解服务、看仪表盘
    CLI 终端敲命令 快速，可脚本化 需要记命令 日常运维、自动化脚本
    SDK 代码调函数 深度集成 需要编程 应用开发、复杂自动化

    CLI（Command Line Interface）即命令行工具，通过终端敲命令操作AWS资源，可理解为"用键盘和AWS对话"。

    1.3 AWS服务分类速览#

    类别 核心服务 用途
    计算 EC2, Lambda, ECS 运行应用程序
    存储 S3, EBS, EFS 存储数据
    数据库 RDS, DynamoDB, ElastiCache 管理数据
    网络 VPC, Route 53, CloudFront 连接和分发
    安全 IAM, KMS, WAF 保护资源
    监控 CloudWatch, X-Ray 观察和调试

    每个服务在CLI中都有对应命令，如aws ec2、aws s3、aws lambda，命令名即服务名。

    2. AWS CLI安装#

    参考文档：

    • Install or update to the latest version of the AWS CLI
    • Uninstall the AWS CLI version 2

    2.1 Windows#

    下载AWS CLI MSI安装程序，运行.msi文件按提示完成安装，打开cmd或PowerShell验证：

    aws --version

    2.2 macOS#

    brew install awscli

    2.3 Linux#

    安装

    curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
aws --version
rm -rf aws awscliv2.zip

    更新

    sudo ./aws/install --bin-dir /usr/local/bin --install-dir /usr/local/aws-cli --update

    卸载

    # 删除符号链接
sudo rm /usr/local/bin/aws
sudo rm /usr/local/bin/aws_completer
# 删除主安装文件夹
sudo rm -rf /usr/local/aws-cli
# 删除配置信息（可选）
rm -rf ~/.aws/

    2.4 Docker#

    # 从 Amazon ECR Public 安装
docker pull public.ecr.aws/aws-cli/aws-cli
# 从 Docker Hub 安装
docker pull amazon/aws-cli

    参考文档：Run the AWS CLI from the official Amazon ECR Public or Docker images

    2.5 CloudShell#

    AWS在网页控制台中提供免费的基于Amazon Linux的CloudShell。由于主机托管在AWS，查询速度可能比本地运行的CLI更快。

    3. AWS CLI配置#

    3.1 配置访问凭证几种方式#

    3.1.1 交互式配置（最常用）#

    $ aws configure
AWS Access Key ID [None]: xxx
AWS Secret Access Key [None]: xxx
Default region name [None]:   # 如 us-east-1 | ap-northeast-1 | ap-east-1
Default output format [None]: # 如 table | json

    获取访问密钥：IAM控制面板

    3.1.2 单项设置#

    aws configure set aws_access_key_id xxx
aws configure set aws_secret_access_key xxx
aws configure set default.region us-east-1
aws configure set default.output json

    3.1.3 环境变量（临时使用）#

    export AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
export AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
export AWS_DEFAULT_REGION=us-west-2

    3.1.4 多环境切换（Named Profiles）#

    aws configure --profile dev    # 配置开发环境
aws configure --profile prod   # 配置生产环境
aws s3 ls --profile prod       # 指定使用生产环境凭证

    3.2 配置文件位置#

    配置文件位于~/.aws目录：

    区域和输出格式配置 ~/.aws/config：

    [default]
region = us-east-1
output = table

    凭证配置 ~/.aws/credentials：

    [default]
aws_access_key_id = xxx
aws_secret_access_key = xxx

    3.3 凭证说明#

    • Access Key ID：以AKIA开头，标识身份（相当于工卡号码）
    • Secret Access Key：证明身份（相当于工卡密码/芯片），创建时只显示一次，之后无法再查看

    4. AWS CLI使用#

    4.1 命令格式#

    所有AWS CLI命令遵循统一格式：

    aws <service> <action> [--options]
    • aws：调用AWS CLI工具
    • service：操作的服务名
    • action：执行的操作
    • --options：附加条件

    示例拆解：

    aws s3 ls --region us-east-1
    部分 值 含义
    CLI工具 aws 调用AWS CLI
    服务名 s3 操作S3服务
    操作 ls 列出内容
    选项 –region us-east-1 指定美东区域

    4.2 命令命名规律#

    操作模式 含义 示例
    create-* 创建资源 create-bucket, create-user
    describe-* 查看资源详情 describe-instances
    list-* 列出资源 list-functions, list-users
    delete-* 删除资源 delete-bucket, delete-user
    update-* 更新资源 update-function-code
    put-* 设置/写入 put-object, put-bucket-policy
    get-* 获取/读取 get-object, get-caller-identity

    4.3 输出格式#

    # JSON格式（默认，适合程序处理）
aws iam list-users --output json

# Table格式（人类友好，适合终端查看）
aws iam list-users --output table

# Text格式（适合Shell脚本用awk/grep处理）
aws iam list-users --output text

    使用建议：

    • 日常查看结果：使用--output table，清晰易读
    • 脚本处理数据：使用--output json配合jq工具

    4.4 Region参数#

    • 不指定--region时，CLI使用默认配置的区域
    • 临时覆盖默认区域：添加--region xx-xxxx-x选项

    选择Region的考量因素：

    • 延迟：选择离用户最近的区域（如服务澳洲用户选ap-southeast-2悉尼）
    • 合规：某些数据必须留在特定地理位置
    • 服务可用性：并非所有服务在所有区域可用
    • 成本：不同区域价格不同

    4.5 常用命令示例#

    4.5.1 确认当前身份#

    aws sts get-caller-identity
    • sts：Security Token Service，AWS身份验证服务
    • get-caller-identity：返回当前账号和用户信息，相当于AWS的whoami命令
    • 排查AWS权限问题时，第一步永远是执行此命令

    4.5.2 列出S3存储桶#

    aws s3 ls

    返回为空表示当前账号下没有任何存储桶，属于正常情况。

    S3的CLI命令设计参考Linux文件操作：ls（列出）、cp（复制）、mv（移动）、rm（删除）、mb（创建桶）、rb（删除桶）。

    4.5.3 创建S3存储桶#

    aws s3 mb s3://my-first-bucket
    • mb：Make Bucket的缩写
    • s3://：S3的URI格式前缀
    • S3存储桶名称需全局唯一，因为S3支持通过URL直接访问文件（如https://my-bucket.s3.amazonaws.com/photo.jpg），名称重复会导致URL冲突
    • 推荐命名规范：公司名-环境-用途，如acme-prod-uploads

    5. 凭证安全#

    5.1 安全红线#

    禁止行为 后果
    把Access Key写在代码里 代码泄露即密钥泄露
    把Access Key提交到Git 公开仓库数分钟内被自动化工具扫描并滥用
    用聊天工具发送密钥 聊天记录被攻击者获取导致密钥泄露
    使用Root账号的Access Key Root权限无限制，被盗后果最严重

    真实案例：某开发者将AWS Access Key提交到公开GitHub仓库，3分钟内被自动化扫描机器人发现，并用于在全球多个Region启动GPU实例挖矿，隔日发现时账单已超$12,000。

    5.2 密钥泄露应急流程#

    • 立即停用泄露的密钥（IAM → Users → Security Credentials → Deactivate）
    • 检查CloudTrail日志，确认是否被滥用
    • 创建新的Access Key
    • 更新所有使用旧密钥的地方
    • 删除旧密钥
    • 如已提交到Git，从历史记录中彻底清除

    5.3 账号安全最佳实践#

    • 启用MFA（多因素认证）：即使密码泄露，没有手机验证码也无法登录
    • 创建IAM用户，不使用root账号：root账号权限无限制，日常操作应使用权限受限的IAM用户
    • 设置账单警报：当费用超过预设阈值时立即通知，第一时间发现异常
    • 启用CloudTrail日志：记录所有API调用，出事时可追溯"谁做了什么"
